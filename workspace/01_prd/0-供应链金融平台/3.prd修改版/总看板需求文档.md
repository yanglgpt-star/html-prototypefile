# 供应链金融平台 - 总看板需求文档

## 1. 需求概述

### 1.1 产品定位
供应链金融平台总看板是面向平台管理层和运营人员的数据可视化仪表盘，提供业务全局视图，支持实时监控核心指标、业务趋势分析和风险预警。

### 1.2 目标用户
| 角色 | 使用场景 | 核心诉求 |
|------|----------|----------|
| 平台高管 | 晨会汇报、月度经营分析 | 一眼掌握业务全局，快速发现异常 |
| 风控总监 | 风险监控、资产质量评估 | 及时发现风险事件，防范损失 |
| 运营经理 | 日常运营监控、任务调度 | 了解业务动态，处理待办事项 |
| 产品经理 | 数据分析、产品优化 | 洞察用户行为，指导产品迭代 |

### 1.3 需求目标
- 构建统一的数据看板入口，替代多系统切换查看
- 实现关键业务指标实时监控，支持5分钟级数据刷新
- 建立风险预警机制，异常事件主动推送
- 支持数据下钻，从概览到明细的穿透分析

## 3. 功能模块详细说明

### 3.1 核心指标栏（KPI Summary）

#### 3.1.1 累计融资额
- **指标定义**：平台自成立以来累计发生的融资放款总额
- **计算公式**：SUM(历史所有放款金额)
- **展示格式**：¥58.68亿（保留两位小数）
- **趋势指标**：
  - 同比：与去年同期的增长率（▲35.2%）
  - 环比：与上期的增长率（▲8.6%）
- **视觉设计**：
  - 数值：28px 粗体深色
  - 上升：绿色箭头+百分比
  - 下降：红色箭头+百分比

#### 3.1.2 融资总余额
- **指标定义**：当前时点所有未结清融资的剩余本金总额
- **计算公式**：SUM(所有在贷订单剩余本金)
- **展示格式**：¥12.58亿
- **数据意义**：反映平台当前资产规模
- **趋势指标**：同比 ▲12.5%，环比 ▲3.2%

#### 3.1.3 人均余额
- **指标定义**：当前在贷客户的平均融资余额
- **计算公式**：融资总余额 / 在贷客户数
- **展示格式**：¥86.5万
- **数据意义**：反映客户质量和大单占比
- **趋势指标**：同比 ▲5.8%，环比 ▼1.2%

#### 3.1.4 累计还款额
- **指标定义**：平台成立以来累计收到的还款总额
- **计算公式**：SUM(历史所有还款金额)
- **展示格式**：¥46.10亿
- **数据意义**：反映资金回流情况
- **趋势指标**：同比 ▲42.6%，环比 ▲6.8%

#### 3.1.5 当年融资额
- **指标定义**：本年度累计发生的融资放款总额
- **计算公式**：SUM(本年1月1日至今的放款金额)
- **展示格式**：¥15.26亿
- **数据意义**：反映当年业务增速
- **趋势指标**：同比 ▲28.4%，环比 ▲5.3%

#### 3.1.6 交互设计
| 交互 | 触发方式 | 效果 |
|------|----------|------|
| 查看详情 | 点击KPI卡片 | 下钻到对应明细列表页 |
| 刷新数据 | 手动刷新按钮 | 重新加载最新数据 |
| 设置阈值 | 管理员权限 | 配置预警阈值 |

### 3.2 业务趋势与构成（Core Business）

#### 3.2.1 融资规模走势
- **图表类型**：混合图表（柱状图+折线图）
- **数据维度**：
  - X轴：时间（月份）
  - Y轴左：月放款金额（柱状，蓝色渐变）
  - Y轴右：融资余额（折线，橙色）
- **时间范围**：近12个月（默认），支持切换近6月/近30天
- **数据说明**：
  - 柱状高度表示当月放款金额
  - 折线点表示月末融资余额
- **交互功能**：
  - 时间维度切换按钮
  - 悬停显示具体数值
  - 点击柱状/折点筛选明细

#### 3.2.2 资产分布-按品类
- **图表类型**：环形图（中心显示总余额）
- **数据维度**：
  - 手机：45%
  - 家电：33%
  - 其他：22%
- **视觉设计**：
  - 不同品类使用不同主题色
  - 右侧显示图例和占比
  - 中心显示总余额数值
- **交互功能**：
  - 悬停扇区放大显示
  - 点击扇区筛选该品类明细

#### 3.2.3 资产分布-按业务类型
- **图表类型**：环形图
- **数据维度**：
  - 代储代销：28%
  - 先货后款：31%
  - 先款后货：25%
  - 撮合业务：16%
- **视觉设计**：与品类分布图表保持风格一致
- **交互功能**：支持点击查看业务类型详情

#### 3.2.4 库存监管总览
- **模块定位**：实时监控库存状态，预警积压风险
- **核心指标**：
  | 指标 | 数值 | 说明 |
  |------|------|------|
  | 监管库存总数 | 28,560台 | 当前在库监管商品总数 |
  | 监管货值 | ¥4.26亿 | 在库商品总货值 |
  | 在库周转天数 | 18.5天 | 平均库存周转天数 |
  | 预警SKU数 | 23个 | 触发预警的SKU数量 |
- **库存状态分布**：
  | 状态 | 占比 | 颜色 |
  |------|------|------|
  | 正常 | 65% | 绿色 #52c41a |
  | 周转慢 | 20% | 黄色 #faad14 |
  | 预警 | 10% | 红色 #ff4d4f |
  | 冻结 | 5% | 灰色 #d9d9d9 |
- **预警规则**：
  - 积压超30天：黄色预警
  - 积压超45天：红色预警
  - 积压超60天：紧急处理
- **交互功能**：
  - 点击预警项跳转详情
  - 点击查看全部库存列表

### 4.3 数据计算逻辑

#### 累计融资额
```sql
SELECT SUM(loan_amount) 
FROM loan_orders 
WHERE status IN ('disbursed', 'repaid', 'overdue')
```

#### 融资总余额
```sql
SELECT SUM(remaining_principal) 
FROM loan_orders 
WHERE status IN ('disbursed', 'overdue')
```

#### 人均余额
```sql
SELECT SUM(remaining_principal) / COUNT(DISTINCT customer_id)
FROM loan_orders 
WHERE status IN ('disbursed', 'overdue')
```

## 5. 非功能需求

### 5.1 性能要求
- 页面首屏加载时间 < 2秒
- KPI指标查询响应时间 < 500ms
- 图表渲染时间 < 1秒

- 支持自定义布局配置

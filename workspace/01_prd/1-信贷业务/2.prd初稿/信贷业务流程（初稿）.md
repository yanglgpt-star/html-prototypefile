## 1. 产品概述

### 1.1 产品背景

信贷业务是供应链金融平台的核心模块，用于管理客户从准入、额度申请、融资申请、还款到解质押的全生命周期业务流程。系统支持资金方、核心企业、融资企业等多方角色的业务协同。

### 1.2 产品定位

为供应链金融业务提供完整的信贷管理解决方案，包括：
- 客户准入管理
- 额度管理
- 融资管理
- 还款管理
- 仓储监管
- 解质押管理

### 1.3 目标用户

| 角色 | 说明 |
|------|------|
| 资金方 | 提供资金支持，审核融资申请 |
| 核心企业 | 供应链中的核心企业，可为其上下游提供担保 |
| 融资企业 | 申请融资的中小企业 |
| 平台运营 | 管理平台业务，审核各类申请 |

## 2. 业务流程

### 2.1 整体业务流程

```
融资需求收集 → 客户准入 → 额度申请 → 融资申请 → 入库申请 → 还款申请 → 解押申请
```

### 2.2 额度管理流程

**额度管理**：
- 给予总额度、期限等（例如：1000万）
- 融资管理：使用额度、额度锁定（使用300万，额度减少300万，剩余额度700万）
- 还款管理：恢复额度（还款100万，额度增加100万，剩余额度800万）

### 2.3 金额-资产关系

**额度**：
- 最多额度（例如：10,000,000.00）
- 最大申请金额（例如：90%）

**融资**：
- 质押货物货值：单价×数量=货值总价（例如：单价10,000.00 × 数量100 = 1,000,000.00）
- 最大申请金额：货值总价×融资比例（例如：900,000.00）
- 本次申请金额：900,000.00
- 额度占用：900,000.00
- 剩余额度：9,100,000.00

**还款**：
- 提货数量：100
- 还款本金：提货数量×单价×融资比例=900,000.00
- 额度占用：0
- 剩余额度：10,000,000.00

## 6. 附录：原始识别内容

### 流程图关键信息

**整体流程**：
- 融资需求收集 → 线下沟通融资需求 → 新增客户信息（资金方、仓储方、监管方、核心企业、融资企业） → 准入申请 → 审核准入信息 → 额度申请 → 审批额度 → 融资申请 → 审核融资申请 → 入库申请 → 入库完成 → 还款申请 → 审核还款信息 → 出库申请 → 出库完成 → 质押状态更新

**额度管理核心逻辑**：
- 额度申请：给予总额度、期限等
- 融资管理：使用额度，额度锁定
- 还款管理：恢复额度

**金额-资产关系**：
- 质押货物货值 = 单价 × 数量
- 最大申请金额 = 货值总价 × 融资比例
- 还款本金 = 提货数量 × 单价 × 融资比例

## 1. 项目背景与目标

### 1.1 需求简介

信贷业务是供应链金融平台的核心模块，用于管理客户从准入、额度申请、融资申请、还款到解质押的全生命周期业务流程。系统支持资金方、核心企业、融资企业、监管企业、仓储企业、平台方等多方角色的业务协同，实现供应链金融业务的数字化管理。

### 1.2 业务诉求

| 序号 | 诉求 | 说明 |
|------|------|------|
| 1 | 客户全生命周期管理 | 从融资需求收集、准入申请、额度管理到融资还款的完整闭环 |
| 2 | 多方角色协同 | 支持资金方、核心企业、融资企业、监管方、仓储方等多方业务协同 |
| 3 | 灵活的额度管理 | 支持循环额度与非循环额度，额度随融资占用、随还款恢复 |
| 4 | 风控审核体系 | 建立多级审核机制，支持平台审核与资方审核 |
| 5 | 质押物监管 | 对入库质押物进行全程监管，支持解质押管理 |

### 1.3 目标用户

| 角色 | 说明 |
|------|------|
| 平台运营 | 管理平台业务，审核各类申请，维护基础数据 |
| 资金方 | 提供资金支持，审核融资申请，管理授信额度 |
| 核心企业 | 供应链中的核心企业，可为其上下游提供担保或信用支持 |
| 融资企业 | 申请融资的中小企业，使用额度进行融资 |
| 监管企业 | 对质押货物进行监管，确保货物安全 |
| 仓储企业 | 提供货物仓储服务，管理入库出库 |

### 2.3 金额-资产关系

```
┌─────────────────────────────────────────────────────────────────┐
│                       金额-资产关系图                             │
└─────────────────────────────────────────────────────────────────┘

    额度
    ├── 最多额度 (例:10,000,000.00)
    │   └── 最大申请金额 = 货值总价 × 融资比例 (例:90%)
    │
    ├── 融资
    │   ├── 质押货物货值
    │   │   ├── 单价: 10,000.00
    │   │   ├── 数量: 100
    │   │   └── 货值总价: 单价×数量 = 1,000,000.00
    │   ├── 最大申请金额 = 货值总价×融资比例 = 900,000.00
    │   ├── 本次申请金额: 900,000.00
    │   ├── 额度占用: 900,000.00
    │   └── 剩余额度: 9,100,000.00
    │
    └── 还款
        ├── 提货数量: 100
        ├── 还款本金 = 提货数量×单价×融资比例 = 900,000.00
        ├── 额度占用: 0
        └── 剩余额度: 10,000,000.00
```

1. 还款状态更新为"已通过"

1. 还款状态更新为"已拒绝"
2. 平台审核状态更新为"已拒绝"
3. 额度不恢复
4. 可重新发起还款申请

##### 3.7.5.5 数据取源规则

2. **质押数量**：融资全部的质押数量（入库时填写的数量）
3. **解质押数量**：还款申请中申请解押数量的汇总
4. **状态计算**：
   - 锁定：质押数量 - 解质押数量 > 0
   - 已解锁：质押数量 - 解质押数量 = 0

**示例**：
- 融资申请A：质押iPhone 15 Pro 100台
- 还款申请1：解押20台 → 剩余80台（锁定状态）
- 还款申请2：解押30台 → 剩余50台（锁定状态）
- 还款申请3：解押50台 → 剩余0台（已解锁状态）

- 千分位分隔符

- 示例：¥100,000.00

### 4.3 日期时间格式

- 日期：YYYY-MM-DD，如：2026-04-01
- 日期时间：YYYY-MM-DD HH:mm:ss，如：2026-04-01 14:30:00

### 4.4 计算规则

| 计算项 | 公式 | 说明 |
|--------|------|------|
| 货值总价 | 单价 × 数量 | 货物总价值 |
| 最大申请金额 | 货值总价 × 融资比例 | 基于货值可融资的最大金额 |
| 还款本金 | 提货数量 × 单价 × 融资比例 | 提取货物对应的应还本金 |
| 剩余额度 | 总额度 - 额度占用 + 还款恢复额度 | 循环额度下还款后恢复 |
| 期限（月） | (截止年 - 生效年) × 12 + (截止月 - 生效月) | 根据生效日期和截止日期计算 |
| 剩余应还金额 | 融资金额 - 已还金额 | 还需还款的金额 |
| 逾期利息 | 逾期本金 × 逾期利率 × 逾期天数 / 360 | 逾期后产生的利息 |

## 1. 项目背景

### 1.1 需求简介

代采销售协议管理系统是供应链金融平台的核心业务模块，用于管理代采销售协议的全生命周期，支持从协议创建、商品管理、合同上传到采购执行和分销发布的完整业务流程。系统为委托方、代采方、供应商三方提供协议管理工具，实现代采垫资业务模式的数字化管理。

### 1.2 业务诉求

- **解决协议管理分散问题**：当前代采协议的创建、审批、执行分散在不同系统或线下处理，缺乏统一管理
- **提升业务协同效率**：委托方、代采方、供应商三方信息共享，减少沟通成本
- **控制资金风险**：通过保证金机制和服务费率设置，控制代采垫资风险
- **实现业务闭环**：从协议签订到采购执行、商品分销的全链路数字化

---

## 2. 业务流程简述

代采销售协议的主流程为：创建代采协议 → 保存/提交 → 等待保证金缴纳 → 上传合同 → 保证金支付 → 协议生效 → 创建采购单 → 采购执行 → 商品分销发布。

协议状态与保证金状态、合同状态相互关联，共同决定协议可执行的操作。

## 3. 详细功能说明

### 3.1 代采协议创建

- **位置**：代采协议管理 → 创建代采销售协议
- **目标**：完成代采协议的基础信息录入、业务条款设置、商品信息添加和附件上传

#### 3.1.1 页面布局

**整体结构**：
- 页面标题栏：显示"代采协议创建"
- 内容区域分为四个卡片式模块：基础信息、业务条款、商品信息、附件信息
- 底部固定操作栏：保存按钮、提交按钮

**模块排列**：
- 基础信息与业务条款采用两列网格布局并排展示
- 商品信息模块占满宽度，位于下方
- 附件信息模块位于最下方

#### 3.1.2 基础信息

**表单字段**：

| 字段名称 | 类型 | 必填 | 校验规则 | 说明 |
|---------|------|------|---------|------|
| 协议名称 | 文本输入 | 是 | 最多50字符，不能为空 | 自定义命名，如"苹果12代采协议" |
| 协议编号 | 系统自动生成 | - | 规则：DCXY + 年月 + 6位随机码 | 示例：DCXY20260900090，创建时自动生成 |
| 代采总额 | 金额显示 | - | 默认0，添加商品后自动带出，等于采购总价汇总 | 格式：千分位分隔，保留两位小数，如"100,000.00" |
| 委托方 | 下拉选择 | 是 | 从委托方档案中选择，枚举值，无仓注册客户 | 如：采购商1、醒市 |
| 代采方 | 下拉选择 | 是 | 下拉选择，枚举值同运营助手-财务管理-预付采购中付款主体的枚举值 | 如：代采方1、能良 |
| 供应商 | 下拉选择 | 是 | 下拉选择，枚举值同运营助手-财务管理-预付采购中供应商的枚举值；选择代采方后，供应商才支持选择，选择代采方的关联方 | 如：供应商1、能良科技 |
| 提货起始日期 | 日期选择 | 是 | 手动填写日期，小于等于提货截止日期 | 协议允许提货的最早日期 |
| 提货截止日期 | 日期选择 | 是 | 手动填写日期，大于等于当前日期 | 协议允许提货的最晚日期 |
| 创建人 | 文本显示 | - | 自动带入当前登录用户姓名 | 如：张三 |
| 创建日期 | 日期显示 | - | 默认当前日期 | 格式：YYYY-MM-DD，如：2026-09-09 |

**交互说明**：
- 选择委托方、代采方、供应商后，系统自动校验三方关联关系是否合法
- 提货截止日期早于提货起始日期时，日期选择器下方红色提示："截止日期必须晚于起始日期"
- 代采总额根据商品信息模块的数据实时计算更新

**异常处理**：
- 下拉选择框数据加载失败时，提示"数据加载失败，请刷新重试"
- 必填项未填写时点击保存或提交，滚动定位至第一个未填项并红色高亮边框

#### 3.1.3 业务条款

**表单字段**：

| 字段名称 | 类型 | 必填 | 校验规则 | 说明 |
|---------|------|------|---------|------|
| 业务模式 | 文本显示 | - | 固定值：代采垫资，不能修改 | 不可编辑，仅展示 |
| 服务费率（日）‰ | 数字输入 | 是 | 0-1000之间的数字，支持小数点后2位 | 按日计算的服务费率，如：0.92‰ |
| 保证金比例% | 数字输入 | 是 | 0-100之间的数字，支持小数点后2位 | 代采总额需缴纳的保证金比例 |
| 保证金金额 | 自动计算 | - | 填写保证金比例后自动带出，保证金比例 × 代采总额，保留两位小数，想上取 | 实时计算显示 |
| 保证金最迟付款日期 | 文本显示 | - | 填写提货起始日期后，自动带出，等于提货起始日期，不能修改 | 不可编辑，仅展示 |

**交互说明**：
- 服务费率输入框失焦时校验格式，不符合规则时清空并提示"请输入0-1000之间的数字"
- 保证金比例输入后，保证金金额实时重新计算
- 保证金金额计算结果保留两位小数，向上取整

**弹窗表单字段**：

| 字段名称 | 类型 | 必填 | 校验规则 | 说明 |
|---------|------|------|---------|------|
| 品牌名称 | 下拉选择 | 是 | 调用商品信息接口，选择品牌名称 | 如：小米、苹果 |
| 商品名称 | 下拉选择 | 是 | 选择品牌后级联加载该品牌下商品 | 如：小米12、苹果17 |
| SKU | 下拉选择 | 是 | 选择商品对应的SKU，选择SKU后自动带出对应的SKU编码 | 如：256G黑色、128G白色 |
| 无仓SKU编码 | 文本显示 | - | 选择SKU后自动带出 | 系统内部编码 |
| 商家SKU编码 | 文本输入 | 是 | 默认：无仓SKU编码，支持修改 | 供应商自定义编码 |
| 采购单价 | 数字输入 | 是 | 手动输入，支持两位小数 | 单件商品采购价格 |
| 采购数量 | 数字输入 | 是 | 手动输入，整数 | 采购件数 |
| 采购总价 | 自动计算 | - | 默认0，采购单价×采购数量，不支持修改 | 实时计算显示 |

**弹窗交互**：
- 品牌、商品、SKU三级级联，上级未选择时下级禁用
- 级联数据加载时显示加载动画
- 采购单价和数量输入后，采购总价实时计算
- 点击"保存"按钮校验所有必填项，校验通过则添加至商品列表并关闭弹窗
- 点击"取消"按钮或右上角关闭图标，直接关闭弹窗不保存

**弹窗异常处理**：
- 商品库数据加载失败时，弹窗内提示"商品数据加载失败，请刷新重试"
- 保存时网络异常，保留弹窗内数据，提示"网络异常，请检查网络后重试"

#### 3.1.5 附件信息

**上传区域**：
- 显示"点击或拖拽文件至此处上传"提示文字
- 支持点击选择文件或拖拽文件上传
- 支持多文件上传
- 支持的文件格式：PDF、DOC、DOCX、JPG、PNG

**已上传文件展示**：
- 文件名展示，如："协议合同.pdf"
- 文件大小显示，如："2.5MB"
- 删除按钮：点击后移除该附件

**上传交互**：
- 文件上传过程中显示进度条
- 上传成功后在列表中展示
- 上传失败时提示失败原因（格式不支持/文件过大/网络异常）

**异常处理**：
- 上传文件格式不支持时提示："不支持的文件格式，请上传 PDF、DOC、DOCX、JPG 或 PNG 格式的文件"
- 上传文件超过大小限制时提示："文件大小超过20MB限制，请压缩后重试"

#### 3.1.6 底部操作按钮

**按钮展示**：

| 按钮名称 | 类型 | 用途 | 交互说明 |
|---------|------|------|---------|
| 保存 | 次要按钮 | 保存代采协议数据 | 保存后协议状态为"保存"，停留在当前页 |

- 点击后按钮显示加载动画，禁用防止重复点击

**提交交互**：
- 点击后校验所有必填项，不通过时滚动定位至第一个错误项
- 校验通过后按钮显示加载动画
- 提交成功：跳转至协议详情页，页面顶部绿色提示"协议提交成功"，同步生成应收保证金数据
- 提交失败：保留在当前页，提示失败原因

| 创建采购 | 创建采购单 | 协议状态是生效、执行中时显示"创建采购"按钮，点击弹框 |
| 创建分销 | 创建分销活动 | 协议状态是执行中时显示"创建分销"按钮，点击弹框 |

**操作流程与状态流转对照表**：

| 序号 | 操作 | 协议状态 | 保证金状态 | 合同状态 | 备注 |
|------|------|---------|-----------|---------|------|
| 1 | 创建协议 | 保存 | 待生效 | 未上传 | - |
| 2 | 提交协议 | 待签署合同 | 待生效 | 未上传 | 提交后代采协议数据，状态变成待签署合同，同步生成一条应收保证金数据 |
| 3 | 合同上传完成 | 待交保证金 | 待支付 | 已上传 | 协议列表点击"上传合同"，上传合同文件，确认后 |
| 4 | 保证金确认收款 | 生效 | 已支付 | 已上传 | 保证金管理列表中，点击"确认收款"确认后 |
| 5 | 预付采购已付款 | 执行中 | 已支付 | 已上传 | 查询订单数据，付款状态：已确认付款 |
| 6 | 保证金处理 | 执行中 | 已抵扣/已退回 | 已上传 | 保证金管理列表：点击"原路退回"或"转货款抵扣"，确认后 |
| 7 | 货物全部销售完成 | 已结清 | 已抵扣/已退回 | 已上传 | 实际拣货数量=销售数量 |
| 8 | 超过提货截止日期 | 已违约 | 已支付 | 已上传 | 当前日期超过了提货截止日期，并且实际拣货数量>销售数量 |

**按钮交互说明**：

| 按钮 | 交互说明 |
|------|---------|
| 详情 | 点击跳转至协议详情页，展示协议完整信息 |
| 编辑 | 跳转至协议编辑页，可修改基础信息、业务条款、商品信息、附件；仅保存状态可编辑 |
| 删除 | 点击后直接删除，提示"是否删除该协议？" |
| 提交 | 将协议从"保存"状态提交为"待签署合同"状态，同步生成应收保证金数据 |
| 撤销 | 将协议从"待签署合同"状态撤销回"保存"状态，并删除保证金数据 |

**操作异常处理**：

### 4.2 状态机定义

#### 4.2.1 协议状态机

| 状态值 | 状态说明 |
|--------|---------|
| 保存 | 创建协议未提交前状态 |
| 待签署合同 | 提交后状态 |
| 待交保证金 | 合同签署或者上传合同后状态 |
| 生效 | 保证金已支付后的状态 |
| 执行中 | 创建采购并且已经付款 |
| 已结清 | 剩余总量=0+当前日期小于等于提货结束日期 |
| 已违约 | 剩余总量大于0，并且当前日期大于提货结束日期 |

**状态流转**：

```
[保存] --提交--> [待签署合同]
[待签署合同] --上传合同--> [待交保证金]
[待签署合同] --撤销--> [保存]
[待交保证金] --保证金确认收款--> [生效]
[生效] --预付采购已付款--> [执行中]
[执行中] --货物全部销售完成--> [已结清]
[执行中] --超过提货截止日期--> [已违约]
```

#### 4.2.2 保证金状态机

| 状态值 | 状态说明 |
|--------|---------|
| 待生效 | 提交协议，未签署合同或未上传合同前，保证金数据状态 |
| 待支付 | 合同签署或上传合同后，保证金数据状态 |
| 已支付 | 保证金确认收款 |
| 已抵扣 | 保证金确认转货款抵扣 |
| 已退回 | 保证金确认原路退回 |
| 无 | 保证金比例是0 |

**状态流转**：

```
[待生效] --提交协议--> [待支付]
[待支付] --确认收款--> [已支付]
[已支付] --转货款抵扣--> [已抵扣]
[已支付] --原路退回--> [已退回]
```

#### 4.2.3 合同状态机

| 状态值 | 状态说明 |
|--------|---------|
| 未上传 | 合同未上传 |
| 已上传 | 线上签署的合同上传到系统 |

**状态流转**：

```
[未上传] --上传合同--> [已上传]
```

## 5. 数据规范

### 5.1 编号规则

| 编号类型 | 规则 | 示例 |
|---------|------|------|
| 协议编号 | DCXY + 年月 + 6位随机码 | DCXY20260900090 |
| 采购单号 | CG + 年月日 + 6位流水号 | CG20260401000001 |
| 分销单号 | FX + 年月日 + 6位流水号 | FX20260401000001 |

### 5.2 金额格式

- 保留两位小数
- 千分位分隔符
- 前缀"¥"符号
- 示例：¥100,000.00

### 5.3 日期时间格式

- 日期：YYYY-MM-DD，如：2026-04-01
- 日期时间：YYYY-MM-DD HH:mm:ss，如：2026-04-01 14:30:00

### 5.4 计算规则

| 计算项 | 公式 |
|--------|------|
| 采购总价 | 采购单价 × 采购数量 |
| 代采总额 | ∑(所有商品采购总价) |
| 保证金金额 | 代采总额 × 保证金比例，保留两位小数，向上取整 |
| 分销总价 | 分销数量 × 集采含税价 |
| 剩余总数量 | 实际抵货总数量 - 锁定总数量 - 实际销售总数量 |
| 集采含税价默认值 | max(采购单价+采购单价×0.2，采购单价+采购单价×服务费率(日)%×(分销付款日期-采购支付日期+1)) |

## 6. 非功能性需求

### 6.1 性能要求

- 页面加载时间 ≤ 3秒
- 列表查询响应时间 ≤ 2秒
- 表单提交响应时间 ≤ 3秒

- 文件上传安全检查（病毒扫描、格式校验）

### 6.3 可靠性要求

- 系统可用性 ≥ 99.5%
- 数据每日全量备份
- 支持断点续传（大文件上传）

## 1. 项目背景与目标

### 1.1 需求简介

先货后款是供应链金融平台的一种业务模式，与代采垫资（先款后货）相对。该模式下，采购商先收到货物，后在约定账期内支付货款。系统需要支持从群接龙模板创建、采购订单确认到分销发布的全流程管理。

### 1.2 业务诉求

| 序号 | 诉求 | 说明 |
|------|------|------|
| 1 | 灵活的业务模式 | 支持先货后款业务，满足不同客户需求 |
| 2 | 全链路管理 | 从模板创建到订单确认、分销发布的完整流程 |
| 3 | 风险控制 | 通过服务费率、账期管理等手段控制资金风险 |
| 4 | 高效运营 | 批量操作、模板复用，提升运营效率 |

### 1.3 目标用户

- **业务人员**：创建模板、确认订单、发布分销
- **财务人员**：核对账款、管理服务费
- **管理人员**：监控业务状态、风险控制

### 3.3 确认订单

#### 3.3.1 功能概述

- **入口**：采购订单列表 → 操作列"确认订单"按钮
- **目标**：确认先货后款订单，设置服务费率和账期，生成确认记录
- **弹框尺寸**：中等（宽度500px）

#### 3.3.2 确认订单弹框

**弹框标题**：确认订单

**确认编码生成规则**：
- 系统自动生成
- 规则：`XHHK` + 采购单号
- 示例：XHHK8ea62f8811d8605

**表单字段**：

| 字段名称 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|---------|------|------|--------|---------|------|
| 确认日期 | 日期显示 | - | 当前日期 | 不可修改 | 订单确认日期 |
| 服务费率（日）‰ | 数字输入 | 是 | - | 0-100，支持两位小数 | 日服务费率，如：2.50 |
| 开始日期 | 日期显示 | - | 付款时间 | 不可修改 | 服务开始日期 |
| 结束日期 | 日期选择 | 是 | 付款时间+2个月 | 必须大于开始日期 | 服务结束日期，可修改 |

**计算规则**：
- **默认结束日期** = 开始日期 + 2个月
- **结束日期限制**：不能小于开始日期
- **服务费用** = 采购总价 × 服务费率（日）‰ × 实际天数

**操作按钮**：

```
集采含税价 = max(
    采购单价 × 1.2,
    采购单价 + 采购单价 × 服务费率(日)‰ × (分销付款日期 - 采购支付日期 + 1)
)
```
- 分销付款日期：发布当日日期
- 结果保留两位小数，向上取整
- 支持修改，但必须大于等于计算默认值

**分销数量生成规则**：
- 分销数量为空或0，不生成分销活动数据

**操作按钮**：

| 按钮名称 | 类型 | 交互说明 |
|---------|------|---------|
| 取消 | 次要按钮 | 取消本次分销活动，关闭弹框 |
| 确定发布 | 主要按钮 | 调用分销活动的批量导入接口，发布分销活动 |

**发布后规则**：
- 发布分销活动后不能重新上架
- 发布的分销活动不能提审
- 生成数据在分销管理中展示
- 发布分销活动后不考虑销售订单也能按照账面需求不考虑销售订单也能按照账面需求（此句原文如此，建议确认）

## 4. 数据关联与计算

### 4.1 实体关系

```
群接龙模板 ──→ 采购订单 ──→ 确认记录 ──→ 分销活动
    │              │              │              │
    └──────────────┴──────────────┴──────────────┘
                      │
                商品信息
```

### 4.2 关键字段关联

| 实体 | 关键字段 | 关联实体 | 关联字段 |
|------|---------|---------|---------|
| 群接龙模板 | 商品编码 | 商品信息 | 商品编码 |
| 采购订单 | 采购单号 | 确认记录 | 采购单号 |
| 确认记录 | 确认编码 | 分销活动 | 确认编码/活动编号 |
| 分销活动 | SKU编码 | 商品信息 | SKU编码 |

### 4.3 计算公式汇总

**1. 采购总价**
```
采购总价 = 采购单价 × 采购数量
```

**2. 剩余数量（实时计算）**
```
剩余数量 = 采购数量 - 冻结数量 - 销售数量
```

**3. 集采含税价**
```
集采含税价 = max(
    采购单价 × 1.2,
    采购单价 + 采购单价 × 服务费率(日)‰ × (分销付款日期 - 采购支付日期 + 1)
)
```

**4. 分销总价**
```
分销总价 = 分销数量 × 集采含税价
```

**5. 服务费用**
```
服务费用 = 采购总价 × 服务费率(日)‰ × 实际天数
```

## 5. 状态流转

### 5.1 订单状态流转

```
┌─────────────────────────────────────────────────────────┐
│                      订单状态流转                        │
└─────────────────────────────────────────────────────────┘

[付款完成] ──→ [未确认] ──→ [已确认/执行中] ──→ [已结清]
                                    │
                                    ▼
                              [已违约]
```

### 5.2 确认状态详细流转

| 状态 | 进入条件 | 退出条件 |
|------|---------|---------|
| 执行中 | 确认订单后默认状态 | 满足结清或违约条件 |
| 已结清 | 剩余数量=0 且 当前日期≤服务结束日期 | 无 |
| 已违约 | 剩余数量>0 且 当前日期>服务结束日期 | 结清后转为已结清 |

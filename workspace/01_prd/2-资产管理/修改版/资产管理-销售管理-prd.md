## 1. 项目背景

### 1.1 需求简介

销售管理是资产管理模块的核心功能，用于管理代采业务中的分销活动和销售订单。系统支持分销活动的发布状态管理、销售订单的数据同步与查询，覆盖从分销活动创建到订单完成的全流程跟踪。

### 1.2 业务诉求

- **统一管理分销活动**：集中管理所有分销活动的发布、失效状态
- **订单数据实时同步**：支持自动定时同步和手动触发同步销售订单数据
- **全流程状态跟踪**：跟踪订单从待发货到已收货的完整生命周期
- **数据查询导出**：支持多维度搜索筛选和数据导出

---

## 2. 业务流程简述

销售管理的主流程为：创建分销活动 → 分销活动发布（状态：发布中） → 活动失效（状态：已下架） → 客户下单生成销售订单 → 订单数据同步 → 跟踪订单状态变化（待发货 → 已发货 → 已收货/已取消）。

## 3. 详细功能说明

### 3.1 页面布局

**整体结构**：
- 页面采用标签页切换设计
- 两个标签页：分销活动、销售订单
- 默认显示"分销活动"标签页

**标签页交互**：
- 点击标签页切换显示对应内容
- 切换时保留各标签页的筛选条件
- 标签页名称旁可显示该页数据条数（可选）

---

### 3.2 分销活动

- **位置**：资产管理 → 销售管理 → 分销活动标签页
- **目标**：展示所有分销活动，支持搜索筛选和导出

| 字段名称 | 类型 | 说明 | 交互规则 |

## 4. 数据规则

### 4.1 分销活动状态规则

| 状态 | 判断条件 | 说明 |
|------|---------|------|
| 发布中 | 当前日期 ≤ 失效日期 | 活动可正常展示和下单 |
| 已下架 | 当前日期 > 失效日期 | 活动已失效，不可下单 |

**状态更新机制**：
- 系统每日自动检查并更新状态
- 创建时默认状态为"发布中"

### 4.2 销售订单数据同步规则

**自动同步**：
- 定时任务：每天晚上10点自动同步一次
- 同步范围：订单状态非"已收货"且非"已取消"的订单
- 数据来源：调用接口获取分销订单数据

**手动同步**：
- 点击"获取分销订单数据"按钮触发
- 立即调用接口同步数据
- 同步成功后更新同步时间并刷新列表

**数据更新逻辑**：
- 新增数据：接口返回的新订单插入列表
- 更新数据：根据销售订单号更新已有数据
- 终止更新：订单状态为"已收货"或"已取消"时不再更新

**同步数据字段**：
- 需求单号
- 销售订单号
- 实际销售数量
- 实际销售总金额
- 付款状态
- 发货时间
- 签收时间
- 订单状态

### 4.3 订单状态流转

```
待发货 → 已发货 → 已收货
   ↓
已取消
```

**状态说明**：

| 状态值 | 说明 | 终止同步 |
|--------|------|---------|
| 待发货 | 订单已创建，等待发货 | 否 |
| 已发货 | 订单已发货，等待签收 | 否 |
| 已收货 | 订单已签收，交易完成 | 是 |
| 已取消 | 订单已取消 | 是 |

## 5. 非功能性需求

### 5.1 性能要求

- 页面加载时间 ≤ 3秒
- 列表查询响应时间 ≤ 2秒
- 数据同步响应时间 ≤ 10秒
- 标签页切换响应时间 ≤ 1秒

### 5.2 安全要求

- 敏感数据（金额信息）传输加密
- 销售数据操作记录审计日志（操作人、时间、内容）
- 数据导出需要相应的权限控制

### 5.3 可靠性要求

- 系统可用性 ≥ 99.5%
- 数据每日全量备份
- 同步失败时支持重试机制

## 1. 项目背景

### 1.1 需求简介

保证金管理是代采销售协议管理系统的重要组成部分，用于管理代采业务中委托方缴纳的履约保证金。系统支持保证金的应收管理、收款确认、退回处理、转货款抵扣等全生命周期管理，确保代采业务资金安全。

### 1.2 业务诉求

- **规范保证金管理**：统一保证金的应收、实收、退回、抵扣流程
- **提升财务效率**：支持批量确认收款、原路退回、转货款抵扣等操作
- **保障资金安全**：每笔保证金操作留痕，支持凭证上传和查询
- **状态透明可视**：保证金状态实时更新，便于业务跟踪

---

## 2. 业务流程简述

保证金管理的主流程为：协议提交生成应收保证金 → 等待线下付款 → 财务确认收款 → 保证金状态变为已支付 → 协议执行结束后保证金退回或转货款抵扣。

保证金状态与协议状态紧密关联，不同状态下支持不同的操作。

### 3.2 确认收款

- **位置**：保证金管理列表 → 操作列"确认收款"按钮
- **目标**：确认收到保证金款项，更新保证金状态为已支付

#### 3.2.1 弹窗结构

**弹窗标题**："确认收款"

**弹窗尺寸**：宽度500像素，高度自适应，居中显示

**弹窗内容**：

| 字段名称 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| 收款金额 | 数字输入 | 是 | 默认带出应收保证金金额，支持修改 |
| 收款日期 | 日期选择 | 是 | 请选择收款日期 |
| 支付流水号 | 文本输入 | 否 | 请输入支付流水号 |
| 收款凭证 | 文件上传 | 否 | 上传文件，支持上传收款凭证 |

**弹窗按钮**：
- 取消按钮：关闭弹窗，不保存
- 确认按钮：提交收款信息

#### 3.2.2 弹窗交互

**字段说明**：
- 收款金额：默认显示应收保证金金额，支持修改，必须大于0
- 收款日期：日期选择器，默认当前日期
- 支付流水号：文本输入，最多50字符
- 收款凭证：点击"上传文件"选择文件，支持的格式：PDF、JPG、PNG，单个文件大小限制10MB

**确认交互**：
- 点击"确认"按钮后校验：
  - 收款金额是否大于0
  - 收款日期是否已选择
- 校验不通过时，在对应字段下方红色提示错误信息
- 校验通过后按钮显示加载动画
- 提交成功：
  - 弹窗关闭
  - 列表中该记录的保证金状态更新为"已支付"
  - 实收金额、实收日期、支付流水号更新
  - 顶部绿色提示"收款确认成功"
  - 关联的代采协议状态同步更新为"生效"
- 提交失败：提示失败原因，保留弹窗状态

**异常处理**：
- 网络异常时提示"网络异常，请检查网络后重试"
- 记录状态已变更时提示"该记录状态已变更，请刷新后重试"

### 4.2 状态机定义

#### 4.2.1 保证金状态机

| 状态值 | 状态说明 | 触发条件 |
|--------|---------|---------|
| 待生效 | 协议已提交，合同未上传，保证金数据已生成 | 代采协议提交后自动生成 |
| 待支付 | 合同已上传，等待线下支付保证金 | 代采协议上传合同后 |
| 已支付 | 保证金已确认收款 | 点击"确认收款"并提交成功后 |
| 已退回 | 保证金已原路退回 | 点击"原路退回"并提交成功后 |
| 已抵扣 | 保证金已转货款抵扣 | 点击"转货款抵扣"并提交成功后 |

**状态流转**：

```
[待生效] --上传合同--> [待支付]
[待支付] --确认收款--> [已支付]
[已支付] --原路退回--> [已退回]
[已支付] --转货款抵扣--> [已抵扣]
```

### 4.3 与代采协议状态关联

| 保证金状态 | 代采协议状态 | 说明 |
|-----------|-------------|------|
| 待生效 | 待签署合同 | 协议已提交，合同未上传 |
| 待支付 | 待交保证金 | 合同已上传，等待支付保证金 |
| 已支付 | 生效/执行中 | 保证金已收，协议生效 |
| 已退回 | 已结清/已违约 | 协议结束，保证金退回 |
| 已抵扣 | 已结清/已违约 | 协议结束，保证金抵扣货款 |

## 5. 数据规范

### 5.1 编号规则

| 编号类型 | 规则 | 示例 |
|---------|------|------|
| 保证金单据编码 | DCBZJ + 年月 + 6位流水号 | DCBZJ202609000023 |

### 5.2 金额格式

- 保留两位小数
- 千分位分隔符
- 前缀"¥"符号
- 示例：¥5,981.00

### 5.3 日期时间格式

- 日期：YYYY-MM-DD，如：2026-08-15
- 日期时间：YYYY-MM-DD HH:mm:ss，如：2026-08-15 14:30:00

## 6. 非功能性需求

### 6.1 性能要求

- 页面加载时间 ≤ 3秒
- 列表查询响应时间 ≤ 2秒
- 表单提交响应时间 ≤ 3秒

### 6.2 安全要求

- 敏感数据（金额信息）传输加密
- 保证金操作记录审计日志（操作人、时间、内容、IP）
- 财务操作需要相应的权限控制

### 6.3 可靠性要求

- 系统可用性 ≥ 99.5%
- 数据每日全量备份

## 1. 项目背景

### 1.1 需求简介

先货后款是供应链金融平台的一种业务模式，与代采垫资（先款后货）相对。该模式下，采购商先收到货物，后在约定账期内支付货款。系统需要支持从群接龙模板创建、采购订单确认到分销发布的全流程管理。

### 1.2 业务诉求

- **灵活的业务模式**：支持先货后款业务，满足不同客户需求
- **全链路管理**：从模板创建到订单确认、分销发布的完整流程
- **风险控制**：通过服务费率、账期管理等手段控制资金风险
- **高效运营**：批量操作、模板复用，提升运营效率

## 5. 计算公式

### 5.1 采购总价
```
采购总价 = 采购单价 × 采购数量
```

### 5.2 服务费用
```
服务费用 = 采购总价 × 服务费率(日)‰ × 实际天数
```

### 5.3 集采含税价
```
集采含税价 = max(采购单价 × 1.2, 采购单价 + 采购单价 × 服务费率(日)‰ × 天数)
```

### 5.4 分销总价
```
分销总价 = 分销数量 × 集采含税价
```

### 5.5 剩余数量
```
剩余数量 = 采购数量 - 冻结数量 - 销售数量
```

## 6. 状态流转

### 6.1 订单状态流转

```
[待导入] ──→ [未确认] ──→ [已确认] ──→ [已分销]
               │              │
               │              ▼
               │         [执行中] ──→ [已结清]
               │              │
               │              ▼
               │         [已违约]
               │
               ▼
          [已取消]
```

### 6.2 确认状态流转

| 当前状态 | 触发条件 | 下一状态 |
|---------|---------|---------|
| 执行中 | 剩余数量=0 | 已结清 |
| 执行中 | 当前日期 > 服务截止日期 | 已违约 |
| 已违约 | 结清欠款 | 已结清 |

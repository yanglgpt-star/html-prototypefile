## 1. 项目背景

### 1.1 需求简介

监管管理是供应链金融平台的库存数据监管模块，用于实时追踪和监控商品的入库、出库及库存状态。系统通过数据同步机制，将无仓系统的库存数据同步至本平台，实现对代采业务全流程的库存可视化监管，确保账实相符，降低库存风险。

### 1.2 业务诉求

- **实时库存监控**：通过数据同步，实时掌握各协议、各仓库的库存动态
- **出入库追溯**：完整记录商品的入库、出库明细，支持全链路追溯
- **账实核对**：通过库存数据与实际业务的核对，确保数据准确性
- **风险预警**：监控异常库存变动，及时发现潜在风险
- **数据导出**：支持库存数据的导出，便于线下分析和审计

## 4. 数据关联关系

### 4.1 数据流向

```
无仓系统 → 数据同步 → 监管管理系统
    ↓
入库数据 / 出库数据 / 库存数据
```

### 4.2 数据关联

| 关联维度 | 关联字段 | 说明 |
|---------|---------|------|
| 协议维度 | 协议编码 | 关联代采协议 |
| 采购维度 | 无仓采购单号 | 关联采购管理 |
| 销售维度 | 无仓销售单号 | 关联销售管理 |
| 商品维度 | 商品编码 | 关联商品信息 |
| 仓库维度 | 仓库名称/分仓编码 | 关联仓库信息 |
| 供应商维度 | 供应商名称 | 关联供应商信息 |

### 4.3 数据一致性

- **入库数据**：入库数量应与采购管理的拣货数量保持一致
- **出库数据**：出库数量应与销售管理的实际销售数量保持一致
- **库存数据**：实时库存应等于累计入库减累计出库

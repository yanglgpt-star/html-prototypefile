## 1. 产品概述

### 1.1 产品背景

采购管理是资产管理模块的核心功能，用于管理代采业务中的采购订单全生命周期，包括采购订单查询、退款申请、串码查看等功能。

### 1.2 产品定位

- 为代采业务提供采购订单管理能力
- 支持采购订单数据同步和查询
- 提供退款申请和串码查看功能

### 1.3 目标用户

- 平台运营人员
- 财务人员
- 仓库管理人员

## 2. 功能需求

### 2.1 采购管理列表

#### 2.1.1 功能说明

展示所有采购订单的列表信息，支持搜索、筛选和操作功能。

#### 2.1.2 页面结构

- 页面标题：采购管理
- 搜索区域：多个筛选条件
- 功能按钮：导出、批量申请退款、获取采购单数据
- 同步时间显示
- 数据表格：分页展示
- 分页器

#### 2.1.3 搜索条件

| 字段名称 | 字段类型 | 说明 |
|----------|----------|------|
| 协议编号 | 下拉选择 | 选择关联的代采协议 |
| 采购单号 | 下拉选择 | 选择采购单号 |
| 供应商 | 下拉选择 | 选择供应商 |
| 付款主体 | 文本输入 | 请输入付款主体 |
| 下单日期 | 日期范围 | 开始日期 至 结束日期 |
| 原始单号 | 文本输入 | 请输入原始单号 |
| 下单人 | 文本输入 | 请输入下单人 |

#### 2.1.4 功能按钮

| 按钮名称 | 说明 |
|----------|------|
| 导出 | 导出当前筛选条件下的采购单数据 |
| 批量申请退款 | 批量选择采购单进行退款申请 |
| 获取采购单数据 | 手动触发从接口同步采购单数据 |

## 3. 数据规则

### 3.1 排序规则

- 下单日期（降序）+ 协议编号（降序）

### 3.2 数据同步规则

- 从接口自动获取订单对应的数据
- 定时任务，每天晚上11点自动同步一次
- 点击"获取采购单数据"按钮可手动同步

### 3.3 字段计算规则

| 字段 | 计算规则 |
|------|----------|
| 待退款总金额 | 汇总全部退款金额 |
| 退款总数量 | 汇总全部退款数量 |
| 申请日期 | 退款申请日期 |

---

## 4. 状态说明

| 状态 | 说明 |
|------|------|
| 待发货 | 采购单已创建，等待发货 |
| 已发货 | 采购单已发货 |

## 5. 按钮显示规则

| 按钮 | 显示条件 |
|------|----------|
| 退款申请 | 完成拣货或协议状态已违约，且未拣货数量>0 |
| 查看串码 | 拣货数量>0 |

---

## 6. 备注说明

1. 自动从接口同步采购单数据
2. 定时任务每天晚上11点同步一次
3. 手动点击"获取采购单数据"可立即同步
4. 退款申请需满足条件：完成拣货或协议已违约，且未拣货数量>0

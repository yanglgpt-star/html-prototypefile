## 1. 产品概述

### 1.1 产品背景

销售管理是资产管理模块的核心功能，用于管理代采业务中的分销活动和销售订单。系统支持分销活动的查询和销售订单的数据同步、查询等功能。

### 1.2 产品定位

- 管理分销活动的发布和状态
- 同步和查询销售订单数据
- 跟踪订单全流程状态

### 1.3 目标用户

- 平台运营人员
- 财务人员
- 销售人员

## 3. 数据规则

### 3.1 数据同步规则

- 根据活动ID获取下单后的数据
- 同步获取分销活动后，从待审核到已发货全流程的订单数据
- 获取分销订单数据：定时任务每天晚上10点更新数据
- 点击按钮立即获取一次数据

### 3.2 字段说明

- 分销订单数据字段：需求单号、销售订单号、实际销售数量、实际销售总金额、需提供数据字段

---

## 4. 状态说明

### 4.1 分销活动状态

| 状态 | 说明 |
|------|------|
| 发布中 | 当前日期小于等于失效日期 |
| 已下架 | 当前日期大于失效日期 |

### 4.2 销售订单状态

| 状态 | 说明 |
|------|------|
| 待发货 | 订单已创建，等待发货 |
| 已发货 | 订单已发货 |
| 已收货 | 订单已签收 |
| 已取消 | 订单已取消 |

---

## 5. 备注说明

1. 创建分销活动后，返回分销活动数据，当前日期大于失效日期，状态字段更新为已下架
2. 同步获取分销活动后，从待审核到已发货全流程的订单数据
3. 定时任务每天晚上10点更新数据
4. 订单状态是已收货或已取消不再更新数据

## 1. 项目背景

### 1.1 需求简介

资产管理是供应链金融平台的核心业务模块，用于管理代采销售协议的全生命周期，涵盖从协议创建、保证金管理、采购执行、退款处理到销售分销的完整业务流程。系统为委托方、代采方、供应商三方提供协议管理工具，实现代采垫资业务模式的数字化管理，确保业务流程闭环和资金安全。

### 1.2 业务诉求

- **解决协议管理分散问题**：统一代采协议的创建、审批、执行流程，避免分散在不同系统或线下处理
- **规范资金管理**：通过保证金机制和服务费率设置，控制代采垫资风险
- **提升业务协同效率**：委托方、代采方、供应商三方信息共享，减少沟通成本
- **实现业务闭环**：从协议签订到采购执行、商品分销、销售回款的全链路数字化
- **数据实时同步**：支持采购订单和销售订单的自动同步与手动同步
- **灵活退款处理**：支持完成拣货或协议违约后的退款申请和确认收款

---

## 2. 业务流程简述

资产管理的主流程为：

**协议流程**：创建代采协议 → 保存/提交 → 等待保证金缴纳 → 上传合同 → 保证金支付 → 协议生效 → 创建采购单 → 采购执行 → 商品分销发布 → 客户下单销售 → 订单完成。

**退款流程**：采购完成拣货或协议违约 → 发起退款申请 → 财务确认收款 → 退款完成。

**保证金流程**：协议提交生成应收保证金 → 上传合同后状态变为待支付 → 财务确认收款 → 保证金状态变为已支付 → 协议执行结束后保证金原路退回或转货款抵扣。

整体业务流程形成闭环：协议管理 → 保证金管理 → 采购管理 → 退款管理 → 销售管理。

## 3. 详细功能说明

### 3.1 代采中心（协议管理）

#### 3.1.1 代采协议创建

- **位置**：代采协议管理 → 创建代采销售协议
- **目标**：完成代采协议的基础信息录入、业务条款设置、商品信息添加和附件上传

**页面布局**：
- 面包屑导航：资产管理 / 代采中心 / 创建代采协议
- 页面标题栏：显示"创建代采协议"及描述文字
- 步骤条：显示四个步骤（基础信息、业务条款、商品信息、附件信息）
- Tab导航：可通过Tab切换四个模块
- 内容区域：根据Tab显示对应模块内容
- 底部固定操作栏：取消按钮、保存草稿按钮、提交协议按钮

**页面流程**：
1. 点击"创建代采协议"按钮，进入创建页面
2. 按顺序填写基础信息 → 业务条款 → 商品信息 → 附件信息
3. 可随时保存草稿，或提交协议
4. 提交后返回协议列表页

| 收款抬头 | 能良 | 文本显示 | - | 自动带出 | 默认：订单对应的采购商 |
| 发布时间 | 立即发布 | 下拉选择 | 是 | 立即发布 | 默认：立即发布 |
| 下架时间 | 当日24点下架 | 下拉选择 | 是 | 当日24点下架 | 默认：当日24点下架 |
| 指定分销商 | 请输入 | 文本输入 | 否 | - | 选填，填写后指定采购商采购 |

**集采含税价计算规则**：
```
集采含税价 = max(
    采购单价 × 1.2,
    采购单价 + 采购单价 × 服务费率(日)‰ × (分销付款日期 - 采购支付日期 + 1)
)
```
- 分销付款日期：发布当日日期
- 结果保留两位小数，向上取整
- 支持修改，但必须大于等于计算默认值

**发布后规则**：
- 发布分销活动后不能重新上架
- 发布的分销活动不能提审
- 生成数据在分销管理中展示

## 5. 数据规范

### 5.1 编号规则

| 编号类型 | 规则 | 示例 |
|---------|------|------|
| 协议编号 | DCXY + 年月 + 6位随机码 | DCXY20260900090 |
| 采购单号 | CG + 年月日 + 6位流水号 | CG20260401000001 |
| 分销单号 | FX + 年月日 + 6位流水号 | FX20260401000001 |
| 保证金单据编码 | DCBZJ + 年月 + 6位流水号 | DCBZJ202609000023 |
| 退款申请编码 | DCTK + 年月 + 6位随机码 | DCTK202309002211 |
| 先货后款确认编码 | XHHK + 采购单号 | XHHK8ea62f8811d8605 |

### 5.2 金额格式

- 保留两位小数
- 千分位分隔符
- 前缀"¥"符号
- 示例：¥100,000.00

### 5.3 日期时间格式

- 日期：YYYY-MM-DD，如：2026-04-01
- 日期时间：YYYY-MM-DD HH:mm:ss，如：2026-04-01 14:30:00

### 5.4 计算规则

| 计算项 | 公式 |
|--------|------|
| 采购总价 | 采购单价 × 采购数量 |
| 代采总额 | ∑(所有商品采购总价) |
| 保证金金额 | 代采总额 × 保证金比例，保留两位小数，向上取整 |
| 分销总价 | 分销数量 × 集采含税价 |
| 剩余总数量 | 实际抵货总数量 - 锁定总数量 - 实际销售总数量 |
| 集采含税价默认值 | max(采购单价+采购单价×0.2，采购单价+采购单价×服务费率(日)%×(分销付款日期-采购支付日期+1)) |
| 未拣货数量 | 采购数量 - 拣货数量 |
| 待退款总金额 | ∑(各商品未拣货数量 × 采购单价) |
| 退款总数量 | ∑(各商品未拣货数量) |
| 先货后款剩余数量 | 采购数量 - 冻结数量 - 销售数量 |
| 先货后款服务费用 | 采购总价 × 服务费率(日)‰ × 实际天数 |
| 先货后款集采含税价 | max(采购单价×1.2，采购单价+采购单价×服务费率(日)‰×(分销付款日期-采购支付日期+1)) |

### 5.5 数据同步规则

**采购订单同步**：
- 自动同步：定时任务每天晚上11点自动同步一次
- 手动同步：点击"获取采购单数据"按钮触发

**销售订单同步**：
- 自动同步：每天晚上10点自动执行数据同步
- 手动同步：点击"获取分销订单数据"按钮触发
- 同步范围：所有未完成（非已收货/已取消）的订单
- 终止更新：订单状态为"已收货"或"已取消"时不再更新

## 6. 非功能性需求

### 6.1 性能要求

- 页面加载时间 ≤ 3秒
- 列表查询响应时间 ≤ 2秒
- 表单提交响应时间 ≤ 3秒
- 数据同步响应时间 ≤ 10秒
- 弹窗打开响应时间 ≤ 1秒
- 文件上传速度 ≥ 2MB/s

### 6.2 安全要求

- 敏感数据（协议金额、保证金信息、金额信息）传输加密
- 操作记录审计日志（操作人、时间、内容、IP）
- 财务操作需要相应的权限控制
- 文件上传安全检查（病毒扫描、格式校验）
- 数据导出需要相应的权限控制

### 6.3 可靠性要求

- 系统可用性 ≥ 99.5%
- 数据每日全量备份
- 支持断点续传（大文件上传）
- 同步失败时支持重试机制

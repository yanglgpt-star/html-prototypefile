## 1. 项目背景与目标

### 1.1 需求简介

代储代销是供应链金融平台的核心业务模块，为供应商提供"先入库、后销售"的供应链金融服务。供应商将货物存入平台指定仓库，平台支付履约保证金，货物通过平台挂牌销售，销售完成后进行清分结算。该模式帮助供应商提前回笼资金，同时平台通过货物监管降低风险。

### 1.2 业务诉求

| 序号 | 诉求 | 说明 |
|------|------|------|
| 1 | 供应商资金提前回笼 | 通过支付履约保证金，帮助供应商在销售前获得资金支持 |
| 2 | 货物全程监管 | 从入库到出库的全流程货物监管，降低平台风险 |
| 3 | 平台挂牌销售 | 支持定向销售和不定向销售两种模式 |
| 4 | 自动清分结算 | 销售完成后自动按比例扣回保证金、服务费、仓储费 |
| 5 | 多方协同 | 供应商、采购商、平台、仓储方、金融方多方业务协同 |

### 1.3 目标用户与菜单结构

| 角色 | 菜单结构 | 说明 |
|------|---------|------|
| 供应商（无仓平台） | **醒市无仓**<br>├─ 供应链服务<br>│  └─ 代储代销<br>└─ 我的订单<br>   └─ 代储代销 | 提供货物并申请代储代销服务的企业 |
| 采购商（市场助手） | 市场助手下单 | 通过平台购买代储代销货物的企业 |
| 金融助手 | **金融助手**<br>├─ 代储代销审核<br>├─ 电子仓单管理<br>├─ 保证金拨付<br>├─ 销售活动发布<br>├─ 销售订单管理<br>├─ 出库指令管理<br>├─ 自动分账<br>├─ 票据中心<br>└─ 提货凭证管理 | 管理代储代销全流程业务 |
| 仓储方 | 复用现有仓储系统 | 提供货物仓储服务，管理入库出库 |
| 平台运营 | 后台管理系统 | 管理平台业务，审核各类申请，维护基础数据 |

### 2.2 交易模式说明

**参与方**：
- **供应商**：提供货物，签署合同，获得保证金
- **醒市无仓**：平台方，提供代储代销服务
- **采购商**：购买代储代销货物
- **仓库**：提供仓储服务，管理货物入库出库

**核心流程**：
1. **准入**：核验货物，签署四套合规合同（《B2B平台综合服务协议》、《履约保证金支付协议》、《仓储质押/监管协议》、《强制回购条款》），办理中登网登记
2. **入仓**：实物核验，仓库完成入库后生成入库清单，金融助手根据入库清单生成电子仓单并锁货
3. **支付保证金**：按T+0实时评估货值，支付不超过70%的保证金至供应商监管账户
4. **发布销售活动**：金融助手发布销售活动，同步至市场助手进行挂牌销售
5. **下单支付**：采购商在市场助手下单，全款支付至第三方支付通道（分账系统）
6. **出库指令**：金融助手识别货款到账后，发布出库/发货指令至仓库
7. **确认收货**：买方确认收货（或物流签收），触发分账指令
8. **自动结算**：系统按预设比例扣回：保证金、服务费、仓储费；余款自动结算至供应商银行账户，各方在线下载/打印结算清单及发票申请

**开票规则**：
- 卖方全额开票给买方
- 平台开具服务费+仓储费给卖方

### 3.2 审核与价值评估模块

#### 3.2.1 实物审核

- **位置**：金融助手 → 代储代销审核 → 实物审核
- **目标**：对供应商申报的货物资质、品牌进行合规性初审

**审核内容**：
1. **货物资质审核**：
   - 审核货物品牌授权书
   - 审核货物质量合格证
   - 审核货物来源合法性

2. **品牌合规性审核**：
   - 核实品牌是否在平台白名单
   - 核实供应商是否有品牌销售授权

3. **审核结果**：
   - 通过：进入行情评估环节
   - 不通过：退回申请，供应商修改后重新提交

#### 3.2.2 行情评估

- **位置**：金融助手 → 代储代销审核 → 行情评估
- **目标**：对接行情系统，根据市场波动给出实时评估价（T+0）

**功能说明**：
1. **行情对接**：
   - 对接第三方行情系统
   - 实时获取货物市场价格

2. **评估规则**：
   - 根据货物品牌、型号、规格获取市场价格
   - 考虑市场波动因素，给出评估价
   - 评估价有效期：T+0（当日有效）

3. **评估结果展示**：
   - 货物市场评估价
   - 建议保证金比例（最高不超过70%）

#### 3.2.3 终审定价

- **位置**：金融助手 → 代储代销审核 → 终审定价
- **目标**：给出最终评估价及保证金比例，并将订单同步至运营助手

**审核表单字段**：

| 字段名称 | 类型 | 必填 | 默认值 | 说明 |
|---------|------|------|--------|------|
| 代储订单编码 | 文本显示 | - | 自动生成 | 规则：DC+年月日+8位随机码 |
| 货物总评估价（元） | 文本输入 | 是 | 行情总价 | 默认为行情总价，支持修改 |
| 保证金比例（%） | 文本输入 | 是 | 70% | 最高不超过70% |
| 保证金金额（元） | 文本显示 | - | 自动计算 | 货物总评估价×保证金比例 |
| 评估有效期 | 日期选择 | 是 | 当前日期 | 评估价有效期 |
| 审核意见 | 文本输入 | 是 | - | 审核意见说明 |

### 3.3 代储管理模块（复用现有功能）

**说明**：此模块复用现有仓储管理系统功能，无需新增开发

#### 3.3.1 入库管理

- **位置**：运营助手/仓库端 → 仓储管理 → 入库管理
- **目标**：接收代储代销订单，完成货物入库验收

**功能说明**：
1. **入库接单**：
   - 金融助手终审通过后，订单自动同步至运营助手/仓库端
   - 仓库接收代储代销订单，准备仓库配额

2. **核验收货**：
   - 供应商发货后，录入物流信息
   - 仓库收到货物后，进行实物验收（品牌、型号、数量、外包装）
   - 确认实际入库数量，生成入库清单

3. **入库清单同步**：
   - 入库完成后，入库清单自动同步至金融助手
   - 金融助手根据入库清单生成电子仓单

**入库状态定义**：

### 3.4 融资与代销交易模块

#### 3.4.1 保证金拨付

- **位置**：金融助手 → 保证金拨付
- **目标**：根据实际入库数量，支付履约保证金至供应商监管账户

**功能说明**：

1. **拨付条件**：
   - 货物已完成入库
   - 电子仓单已生成
   - 审核定价已完成

2. **拨付计算**：
   - 保证金金额 = 实际入库货值 × 保证金比例
   - 实际入库货值 = 实际入库数量 × 评估单价
   - 保证金比例最高不超过70%

3. **拨付流程**：
   - 系统生成保证金拨付申请
   - 财务审核通过后，通过平安钱包支付至供应商监管账户
   - 供应商收到保证金后，代储代销业务正式生效

#### 3.4.2 销售模式设定

- **位置**：金融助手 → 销售活动发布 → 销售模式设定
- **目标**：设定"定向销售"或"不定向销售"策略

**销售模式说明**：

1. **定向销售**：
   - 供应商指定特定的采购商购买货物
   - 货物仅对指定采购商可见
   - 适用于供应商已有确定买家的情况

2. **不定向销售（公开销售）**：
   - 货物对所有平台采购商开放
   - 采购商可自由下单购买
   - 适用于供应商希望快速出货的情况

**模式切换规则**：
- 代储代销订单创建时，默认为"不定向销售"
- 供应商可在货物销售前切换销售模式
- 一旦有采购商下单，销售模式不可再修改

### 3.5 履约结算与票据模块

#### 3.5.1 出库履约

##### 3.5.1.1 出库/发货指令发布（金融助手）

- **位置**：金融助手 → 出库指令管理
- **目标**：识别货款到账后，发布出库/发货指令

**功能说明**：

1. **货款到账识别**：
   - 系统实时监测采购商支付状态
   - 确认货款到账后，触发出库流程

2. **出库/发货指令发布**：
   - 金融助手生成出库/发货指令
   - 指令包含：销售订单信息、提货数量、提货码、收货地址等
   - 指令同步至运营助手/仓库端

3. **电子提货码生成**：
   - 生成电子提货码，作为提货凭证
   - 提货码同步至采购商端和仓库端

##### 3.5.1.2 出库执行（运营助手/仓库端-复用现有功能）

- **位置**：运营助手/仓库端 → 仓储管理 → 出库执行
- **目标**：接收出库指令，完成货物出库

**功能说明**：

1. **出库指令接收**：
   - 接收金融助手发布的出库/发货指令
   - 查看出库任务详情

2. **货物解锁**：
   - 仓库确认出库指令后，对相应货物进行解锁
   - 解锁后的货物可正常出库

3. **出库操作**：
   - 仓库根据出库指令，完成货物拣货、包装
   - 生成物流单，安排发货
   - 出库完成后，更新库存状态，通知金融助手

**出库状态定义**：

| 状态值 | 说明 | 流转条件 |
|--------|------|---------|
| 待出库 | 出库指令已生成，等待仓库处理 | 货款到账后 |
| 出库中 | 仓库正在处理出库 | 仓库开始拣货 |
| 已出库 | 货物已发出 | 物流单生成后 |
| 已签收 | 采购商已确认收货 | 采购商确认或物流签收后 |

#### 3.5.2 自动分账

- **位置**：金融助手 → 自动分账
- **目标**：采购商确认收货后，系统自动按比例扣回各项费用

**分账规则**：

1. **触发条件**：
   - 采购商确认收货，或物流自动签收
   - 触发分账指令

2. **分账比例**：
   - 保证金扣回：按实际销售比例扣回已拨付保证金
   - 服务费：按订单金额的一定比例收取
   - 仓储费：按货物存储天数计算
   - 余款：扣除上述费用后，结算给供应商

3. **分账计算示例**：
   - 销售订单金额：100,000元
   - 应扣回保证金：70,000元（假设保证金比例70%）
   - 服务费：1,000元（假设费率1%）
   - 仓储费：500元
   - 供应商实收：28,500元

#### 3.5.3 票据中心

- **目标**：管理各方发票开具与结算清单下载

**功能说明**：

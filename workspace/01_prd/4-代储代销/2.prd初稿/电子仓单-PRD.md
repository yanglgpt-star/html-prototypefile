# 电子仓单功能需求文档

## 1. 项目背景

### 1.1 需求简介
将"动态电子仓单"从业务概念转化为可落地的系统功能，建立"状态驱动+资产账本化"的管理机制，实现仓单全生命周期的数字化管理，支撑代储代销业务模式下货物资产的质押、释放与核销流转。

### 1.2 业务诉求
- **解决货权不清问题**：传统纸质仓单易伪造、难追溯，电子仓单通过系统固化货权归属
- **支撑动态库存管理**：代储代销模式下，货物分批销售、逐步释放，需要仓单状态实时联动
- **满足金融风控要求**：质押状态可视化，确保质权人对质押货物的实时监控能力
- **提升操作效率**：自动生成、自动状态流转，减少人工操作和线下对账成本

---

## 2. 业务流程简述

供应商完成实物入库后，系统根据入库数据自动生成电子仓单（初始状态为"普通存单"）。当平台向供应商拨付保证金后，仓单状态变更为"已质押"，货物进入锁定状态。采购商下单并部分付款后，对应数量货物从仓单中释放，状态变更为"部分释放"并生成电子提货码。当最后一批货物出库完成后，仓单状态变更为"已核销"，流程闭环。

## 3. 详细功能说明

### 3.1 仓单列表

#### 3.1.1 入口与布局
- **位置**：平台后台 → 资产管理 → 电子仓单
- **页面结构**：
  - 顶部：统计卡片区域（总仓单数、质押中、已核销）
  - 中部：筛选查询区域
  - 主体：仓单数据表格

#### 3.1.2 统计卡片
| 卡片名称 | 数据来源 | 交互 |
|---------|---------|------|
| 总仓单数 | 全量仓单计数 | 点击后列表展示全部状态 |
| 质押中 | 状态为"已质押"的仓单计数 | 点击后列表筛选质押中 |
| 已核销 | 状态为"已核销"的仓单计数 | 点击后列表筛选已核销 |

#### 3.1.3 筛选条件
| 筛选项 | 类型 | 说明 |
|-------|------|------|
| 仓单编号 | 文本输入 | 精确匹配，支持扫码输入 |
| 仓单状态 | 下拉选择 | 选项：全部、待审、普通存单、已质押、部分释放、已核销 |
| 存货人 | 文本输入 | 供应商名称模糊搜索 |
| 货物名称 | 文本输入 | 货物名称模糊搜索 |
| 签发日期 | 日期范围 | 开始日期至结束日期 |

#### 3.1.4 列表展示
- **默认排序**：按签发时间倒序
- **分页**：每页20条，支持下拉加载更多
- **空状态**：显示"暂无仓单记录"插画，提供"去创建"快捷入口（跳转入库页面）

**列表字段**：
| 字段 | 宽度 | 说明 |
|-----|------|------|
| 仓单编号 | 140px | 蓝色可点击文本，点击进入详情 |
| 状态标签 | 90px | 不同状态显示不同颜色标签 |
| 存货人 | 150px | 供应商名称，超出截断显示"..." |
| 货物名称 | 120px | 主要货物名称，如有多种显示"等{N}种" |
| 入库数量 | 90px | 右对齐，千分位分隔 |
| 质押金额 | 120px | 右对齐，¥符号+千分位分隔 |
| 签发日期 | 100px | 格式：YYYY-MM-DD |
| 操作 | 100px | 查看、下载按钮 |

**状态标签样式**：
| 状态 | 标签文案 | 颜色 |
|-----|---------|------|
| 0-待审 | "待审核" | 灰色 |
| 1-普通存单 | "普通存单" | 蓝色 |
| 2-已质押 | "已质押" | 紫色 |
| 3-部分释放 | "部分释放" | 橙色 |
| 4-已核销 | "已核销" | 绿色 |

#### 3.1.5 操作按钮
- **查看**：点击后新开页面打开仓单详情
- **下载**：点击后下载该仓单的PDF文件
  - 按钮状态：仓单状态为"待审"时置灰不可点击
  - 点击后显示加载动画，3秒后自动开始下载
  - 下载失败时顶部轻提示："下载失败，请稍后重试"

### 3.2 仓单详情

#### 3.2.1 入口与布局
- **入口**：仓单列表点击仓单编号、或业务通知点击链接
- **页面结构**：
  - 顶部：仓单标题与状态标识
  - 左侧主体：仓单核心信息展示
  - 右侧边栏：操作日志、关联信息

#### 3.2.2 头部信息区
- **仓单标题**：显示"电子仓单"+仓单编号
- **状态标识**：大字状态标签，位于右上角
- **快速操作**：下载PDF、分享链接按钮

#### 3.2.3 基础信息卡片
展示字段：
- 仓单编号（可复制）
- 签发日期
- 有效期至
- 当前状态
- 存货人（点击可跳转供应商详情）
- 保管人（仓库名称）
- 权利人（质权人）

#### 3.2.4 货物明细卡片
- **展示形式**：表格形式展示
- **字段**：货物名称、品牌规格、计量单位、入库数量、入库重量、等级质量、批次炉号、评估单价
- **汇总信息**：
  - 货物总价值（红色大字突出显示）
  - 评估基准日
  - 质检报告编号（可点击跳转质检详情）

**动态展示规则**：
- 当仓单状态为"已质押"或"部分释放"时，显示质押信息栏：
  - 质押金额
  - 质押比例
  - 中登网登记号

- 当仓单状态为"部分释放"时，额外显示释放进度：
  - 原始数量
  - 已释放数量（绿色）
  - 锁定中数量（橙色）
  - 进度条可视化展示

#### 3.2.5 合同关联区
- **展示形式**：卡片列表
- **内容**：与该仓单关联的全部合同编号
- **交互**：点击合同编号可跳转至合同预览页面
- **状态标识**：每个合同显示签署状态（待签/已签/已失效）

#### 3.2.6 变更记录（时间轴）
- **展示形式**：垂直时间轴
- **记录内容**：
  - 时间戳（精确到秒）
  - 操作类型（创建/质押锁定/部分释放/完全核销）
  - 操作人
  - 变更详情（如：释放20吨，剩余锁定80吨）

### 3.3 仓单生成

#### 3.3.1 触发时机
- **自动触发**：WMS系统上报实物入库完成且质检通过后
- **人工触发**：运营人员在入库审核页面点击"生成仓单"

#### 3.3.2 生成规则
- 一个入库批次对应一张仓单
- 仓单编号规则：EWR-YYYYMMDD-四位流水号（如：EWR-20231027-0001）
- 有效期默认：签发日期起365天
- 初始状态：普通存单（Status=1）

#### 3.3.3 前置校验
生成仓单前必须满足以下条件，否则阻断并提示：
| 校验项 | 失败提示 |
|-------|---------|
| 入库质检已通过 | "质检未完成，无法生成仓单" |
| 合同已签署完成 | "监管协议未签署，无法生成仓单" |
| 无重复仓单 | "该入库批次已存在仓单，请勿重复创建" |

### 3.4 状态流转与业务联动

#### 3.4.1 状态定义
| 状态码 | 状态名称 | 业务含义 |
|-------|---------|---------|
| 0 | 待审 | 仓单创建中，尚未生效 |
| 1 | 普通存单 | 已生效，可作为普通凭证，未质押 |
| 2 | 已质押 | 已设定质押，货物全部锁定 |
| 3 | 部分释放 | 部分货物已释放，剩余仍质押中 |
| 4 | 已核销 | 货物全部出库，仓单失效 |

#### 3.4.2 流转规则

**普通存单 → 已质押**
- **触发条件**：平台向供应商拨付保证金成功
- **系统动作**：
  - 仓单状态变更为"已质押"
  - 全部数量计入锁定
  - 向WMS发送货位锁定指令
  - 生成中登网登记记录（待登记）
- **页面反馈**：顶部轻提示"仓单已质押，货物已锁定"

**已质押 → 部分释放**
- **触发条件**：采购商支付部分货款，对应数量货物需要释放
- **系统动作**：
  - 计算释放数量
  - 扣减锁定数量，增加释放数量
  - 状态变更为"部分释放"
  - 生成电子提货码
  - 向WMS发送解锁指令（对应数量）
- **页面反馈**：显示释放成功弹窗，告知已生成提货码

**部分释放 → 已核销**
- **触发条件**：最后一批货物出库完成（提货码全部核销）
- **系统动作**：
  - 仓单状态变更为"已核销"
  - 归档仓单数据
  - 向WMS确认库存扣减
- **页面反馈**：轻提示"仓单已完成核销"

#### 3.4.3 异常处理
- **WMS锁定失败**：
  - 状态回滚为"普通存单"
  - 通知运营人员人工介入
  - 记录异常日志
- **中登网登记失败**：
  - 仓单状态保持"已质押"
  - 标记登记状态为"待补录"
  - 每日提醒运营人员处理

### 3.5 仓单拆分与释放

#### 3.5.1 业务场景
当100吨货物已质押，其中20吨被采购商购买并付款后，需要将这20吨从原仓单中释放。

#### 3.5.2 实现逻辑
采用"主从结构"，不删除原仓单，而是派生释放凭证：
- **主仓单**：保留原仓单编号，记录剩余锁定数量
- **释放子单**：生成新的提货凭证，记录本次释放数量

#### 3.5.3 数据更新规则
- 主仓单锁定数量 = 原锁定数量 - 释放数量
- 主仓单释放数量 = 原释放数量 + 释放数量
- 生成子单记录，关联主仓单ID

#### 3.5.4 展示规则
- 仓单详情页显示"质押总额"、"已释放"、"待提货"
- 点击"已释放"可查看全部释放子单列表
- 每个释放子单显示：释放数量、释放时间、关联订单号、提货码状态

### 3.6 电子仓单PDF

#### 3.6.1 生成时机
- 仓单状态变更时自动重新生成
- 用户在列表或详情页点击下载时实时生成

#### 3.6.2 文件命名规则
{仓单编号}_{状态}_{日期}.pdf
示例：EWR-20231027-0001_已质押_20231027.pdf

#### 3.6.3 内容结构
PDF包含以下章节，与页面展示保持一致：
1. 头部：仓单标题、状态标识、二维码
2. 基础信息：仓单编号、日期、主体信息
3. 货物明细：表格形式展示，含价值汇总
4. 仓储信息：存放位置、保险信息
5. 权利约束：质押信息、限制说明
6. 验证信息：电子签章、区块链存证哈希
7. 底部：签章区域、查验链接

#### 3.6.4 动态水印
根据仓单状态在PDF背景叠加半透明水印：
| 状态 | 水印文字 | 颜色 |
|-----|---------|------|
| 普通存单 | 无 | - |
| 已质押 | "已质押" | 红色 |
| 部分释放 | "部分释放" | 橙色 |
| 已核销 | "已核销" | 灰色 |

---

### 3.7 关联功能

#### 3.7.1 合同关联
- **关联时机**：仓单生成时自动关联该供应商该批次的全部有效合同
- **展示位置**：仓单详情页合同卡片区
- **跳转逻辑**：点击合同编号新开页面打开合同预览

#### 3.7.2 提货码关联
- **数据关联**：提货码表必须记录关联的仓单ID
- **核销联动**：仓库扫码核销提货码时，同步更新仓单释放数量
- **状态校验**：只有提货码状态为"已提货"，才算完成该部分的物理核销

## 5. 附录

### 5.1 数据字段说明

**仓单主表关键字段**：
| 字段名 | 类型 | 说明 |
|-------|------|------|
| 仓单编号 | 字符串 | 唯一标识，EWR前缀 |
| 状态 | 枚举 | 0待审/1普通存单/2已质押/3部分释放/4已核销 |
| 入库总量 | 数值 | 原始入库数量 |
| 锁定数量 | 数值 | 当前质押锁定中的数量 |
| 释放数量 | 数值 | 已售出释放的数量 |
| 关联合同列表 | 数组 | 关联的合同编号集合 |

### 5.2 错误码说明
| 错误码 | 说明 | 处理建议 |
|-------|------|---------|
| EWR001 | 仓单不存在 | 检查仓单编号是否正确 |
| EWR002 | 状态不允许操作 | 确认当前状态是否支持该操作 |
| EWR003 | WMS联动失败 | 联系仓库管理员确认 |
| EWR004 | 合同未签署 | 先完成合同签署流程 |

| 功能 | 供应商 | 运营人员 | 财务人员 | 仓库人员 |

| 查看仓单详情 | 仅自己的 | 全部 | 全部 | 仅所属仓库 |
| 下载PDF | 仅自己的 | 全部 | 全部 | 仅所属仓库 |
| 触发质押 | 否 | 否 | 是 | 否 |
| 触发释放 | 否 | 否 | 是 | 否 |
| 核销确认 | 否 | 否 | 否 | 是 |

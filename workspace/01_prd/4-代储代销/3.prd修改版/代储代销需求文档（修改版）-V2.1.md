## 1. 项目背景与目标

### 1.1 需求简介

代储代销是供应链金融平台的核心业务模块，为供应商提供"先入库、后销售"的供应链金融服务。供应商将货物存入平台指定仓库，平台支付履约保证金，货物通过平台挂牌销售，销售完成后进行清分结算。

### 1.2 业务诉求

| 序号 | 诉求 | 说明 |
|------|------|------|
| 1 | 供应商资金提前回笼 | 通过支付履约保证金，帮助供应商在销售前获得资金支持 |
| 2 | 货物全程监管 | 从入库到出库的全流程货物监管，降低平台风险 |
| 3 | 平台挂牌销售 | 支持定向销售模式 |
| 4 | 自动清分结算 | 销售完成后自动按比例扣回保证金、服务费、仓储费 |
| 5 | 多方协同 | 供应商、采购商、平台、仓储方、金融方多方业务协同 |

### 1.3 目标用户与菜单结构

| 角色 | 菜单结构 | 说明 |
|------|---------|------|
| 供应商（无仓平台） | **醒市无仓**<br>├─ 供应链服务<br>│  ├─ 代储代销<br>│  └─ 仓储回购通（即将上线）<br>├─ 服务管理<br>└─ 我的订单<br>   └─ 代储代销 | 提供货物并申请代储代销服务的企业 |
| 采购商（市场助手） | 市场助手下单 | 通过平台购买代储代销货物的企业 |
| 金融助手 | **金融助手**<br>├─ 代储代销审核<br>├─ 电子仓单管理<br>├─ 保证金拨付<br>├─ 代储订单管理<br>├─ 销售订单管理<br>├─ 出库指令管理<br>├─ 自动分账<br>├─ 票据中心<br>└─ 提货凭证管理 | 管理代储代销全流程业务 |
| 仓储方 | 复用现有仓储系统 | 提供货物仓储服务，管理入库出库 |

## 2. 业务流程与架构

### 2.1 整体业务流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        代储代销整体流程                           │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   注册与准入       │
                    │ (复用现有注册功能 │
                    │  +服务选择+协议)   │
                    └──────────────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   审核与价值评估   │
                    │ (实物审核+行情评估 │
                    │  +终审定价)        │
                    └──────────────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   代储管理         │
                    │ (复用现有仓储功能 │
                    │  +金融仓单管理)    │
                    └──────────────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   融资与代销交易   │
                    │ (保证金拨付+销售   │
                    │  订单管理)         │
                    └──────────────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   履约结算与票据   │
                    │ (出库指令+自动分账 │
                    │  +票据管理)        │
                    └──────────────────┘
```

### 2.2 核心流程

1. **准入**：核验货物，勾选《代储代销服务协议》，办理中登网登记
2. **入仓**：实物核验，仓库完成入库后生成入库清单，金融助手根据入库清单生成电子仓单并锁货
3. **支付保证金**：按T+0实时评估货值，支付不超过70%的保证金至供应商平安钱包账户
4. **销售**：金融助手在代储订单管理中进行定向销售
5. **下单支付**：采购商在市场助手下单，全款支付至第三方支付通道
6. **出库指令**：金融助手识别货款到账后，发布出库/发货指令至仓库
7. **确认收货**：买方确认收货（或物流签收），触发分账指令
8. **自动结算**：系统按预设比例扣回：保证金、服务费、仓储费；余款自动结算至供应商银行账户

## 3. 详细功能说明

### 3.1 电子仓单管理

#### 3.1.1 仓单状态定义

| 状态码 | 状态名称 | 触发事件 | 系统动作 | 权限控制 | UI表现 |
|-------|---------|---------|---------|---------|--------|
| 0 | 待生效(Pending) | 供应商提交货物信息并签署合同 | 仓单记录已创建，但缺乏物理核验，处于"草稿"态 | 仅供应商可见，不可融资，不可挂牌 | 虚线边框，置灰显示 |
| 1 | 普通存单(Active) | 运营助手完成WMS核实验收 | WMS回传确认数据，系统正式签发仓单 | 货权确立，可作为抵押物发起融资申请 | 绿色"已核验"标签 |
| 2 | 已质押(Collateralized) | 金融助手完成保证金拨付 | 系统自动在仓单上打质押锁，同步指令给WMS锁定货位 | 禁止出库，禁止调拨，货权受限 | 红色"已质押"大水印 |
| 3 | 部分释放(Released) | 市场助手识别采购商已支付对应货款 | 计算货款覆盖的货物比例，释放对应额度的锁定状态 | 解冻对应比例的库存，允许仓库凭码发货 | 橙色"部分释放"标签 |
| 4 | 已核销(Closed) | 仓库端扫码核销完毕，最后一笔货离库 | 资产账本结清，仓单生命周期结束 | 状态只读，不可再操作 | 灰色"已结清"水印 |

## 4. 数据规范

### 4.1 编号规则

| 模块 | 名称 | 规则 | 示例 |
|------|------|------|------|
| 代储代销 | 申请编号 | SQ+年月日+8位随机码 | SQ20250909ABCD1234 |
| 代储代销 | 代储订单编号 | DC+年月日+8位随机码 | DC20250909ABCD1234 |
| 仓储管理 | 入库清单编号 | RK+年月日+8位随机码 | RK20250909ABCD1234 |
| 仓储管理 | 仓单编号 | CD+年月日+8位随机码 | CD20250909ABCD1234 |
| 保证金 | 拨付编号 | BF+年月日+8位随机码 | BF20250909ABCD1234 |
| 销售活动 | 活动编号 | HD+年月日+8位随机码 | HD20250909ABCD1234 |
| 销售订单 | 销售订单编号 | XS+年月日+8位随机码 | XS20250909ABCD1234 |

### 4.2 计算规则

| 计算项 | 公式 | 说明 |
|--------|------|------|
| 货物总价 | 单价 × 数量 | 货物总价值 |
| 保证金金额 | 实际入库货值 × 保证金比例 | 最高不超过70% |
| 服务费 | 订单金额 × 服务费率 | 按订单金额比例收取 |
| 仓储费 | 存储天数 × 日仓储费率 × 货物数量 | 按天计算 |
| 供应商实收 | 订单金额 - 扣回保证金 - 服务费 - 仓储费 | 供应商最终收到金额 |

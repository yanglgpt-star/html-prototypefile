## 1. 项目背景与目标

### 1.1 需求简介

代储代销是供应链金融平台的核心业务模块，为供应商提供"先入库、后销售"的供应链金融服务。供应商将货物存入平台指定仓库，平台支付履约保证金，货物通过平台挂牌销售，销售完成后进行清分结算。

### 1.2 业务诉求

| 序号 | 诉求 | 说明 |
|------|------|------|
| 1 | 供应商资金提前回笼 | 通过支付履约保证金，帮助供应商在销售前获得资金支持 |
| 2 | 货物全程监管 | 从入库到出库的全流程货物监管，降低平台风险 |
| 3 | 平台挂牌销售 | 支持定向销售模式 |
| 4 | 自动清分结算 | 销售完成后自动按比例扣回保证金、服务费、仓储费 |
| 5 | 多方协同 | 供应商、采购商、平台、仓储方、金融方多方业务协同 |

### 1.3 目标用户与菜单结构

| 角色 | 菜单结构 | 说明 |
|------|---------|------|
| 供应商（无仓平台） | **醒市无仓**<br>├─ 供应链服务<br>│  ├─ 代储代销<br>│  └─ 仓储回购通（即将上线）<br>├─ 服务管理<br>└─ 我的订单<br>   └─ 代储代销 | 提供货物并申请代储代销服务的企业 |
| 采购商（市场助手） | 市场助手下单 | 通过平台购买代储代销货物的企业 |
| 金融助手 | **金融助手**<br>├─ 代储代销审核<br>├─ 电子仓单管理<br>├─ 保证金拨付<br>├─ **代储订单管理**<br>├─ 销售订单管理<br>├─ 出库指令管理<br>├─ **清结算中心**（原自动分账）<br>├─ 票据中心<br>└─ 提货凭证管理 | 管理代储代销全流程业务 |
| 仓储方 | 复用现有仓储系统 | 提供货物仓储服务，管理入库出库 |

## 2. 业务流程与架构

### 2.1 整体业务流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        代储代销整体流程                           │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   1.注册与准入      │
                    │ (账号注册+协议签署) │
                    └──────────────────┘
                              │

                    ┌──────────────────┐
                    │   2.审核定价       │
                    │ (实物审核+行情评估  │

                    ┌──────────────────┐
                    │   3.入库与仓单     │
                    │ (WMS入库+生成      │

                    └──────────────────┘

                    ┌──────────────────┐
                    │   4.保证金拨付     │
                    │ (支付70%保证金     │
                    │  至平安钱包)        │
                    └──────────────────┘
                              │
                              ▼
                    ┌──────────────────┐
                    │   5.定向销售       │

                    │ (出库指令+自动     │
                    │  分账+票据管理)     │
                    └──────────────────┘
```

### 2.2 核心业务流程闭环

1. **准入阶段**：供应商提交申请 → 平台审核定价 → 生成代储订单
2. **入库阶段**：仓库实物验收 → 生成入库清单 → 金融助手生成电子仓单
3. **融资阶段**：金融助手确认入库 → 支付保证金（70%）至供应商平安钱包
4. **销售阶段**：金融助手在代储订单管理中创建定向销售订单 → 采购商下单支付
5. **履约阶段**：货款到账 → 发布出库指令 → 仓库发货 → 自动分账结算

   - 货物总评估价=所有货物的评估总价之和，不支持修改
   - 保证金=货物总评估价×保证金比例，支持人工修改保证金比例，修改后重新计算保证金金额

### 4.2 计算规则

| 计算项 | 公式 | 说明 |
|--------|------|------|
| 货物总价 | 单价 × 数量 | 货物总价值 |
| 申请总价 | 申请单价 × 数量 | 供应商申请总价值 |
| 行情总价 | 行情单价 × 数量 | 行情系统获取的实时价格总和 |
| 评估总价 | 评估单价 × 数量 | 评估单价可人工调整 |
| 货物总评估价 | ∑评估总价 | 所有货物评估总价之和 |
| 保证金金额 | 货物总评估价 × 保证金比例 | 最高不超过70% |
| 服务费 | 评估总价 × 服务费率（‱） | 按评估总价比例收取，如：评估总价 × 1% |
| 仓储费 | 元/台/天 × 存储天数 × 货物数量 | 按天计算，如：10元/天/台 × 50天 × 10台 |
| 供应商实收 | 销售总额 - 扣回保证金 - 服务费 - 仓储费 | 供应商最终收到金额 |

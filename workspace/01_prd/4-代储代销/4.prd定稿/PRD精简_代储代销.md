## 背景与目标

### 问题现状

**业务侧**：
- 供应商资金回笼周期长，需要先入库、后销售的供应链金融支持
- 货物监管缺乏系统化手段，平台风险管控不足

**系统侧**：
- 缺少完整的代储代销业务流程闭环（准入→审核→入库→融资→销售→结算）
- 多方协同（供应商、采购商、平台、仓储方、金融方）缺乏统一平台支撑

### 需求目标

为供应商提供先入库、后销售的供应链金融服务：

| 目标 | 说明 | 指标 |
|-----|------|------|
| 资金提前回笼 | 入库即可申请70%保证金拨付 | 【待补充量化指标】 |
| 货物全程监管 | 从入库到出库全流程跟踪 | 仓单状态实时可查 |
| 平台挂牌销售 | 支持定向销售模式 | 【待补充量化指标】 |
| 自动清分结算 | 销售完成后自动分账 | 结算准确率100% |

## 范围决策

### 本期包含（做什么）

| 模块 | 范围 |
|-----|------|
| 代储代销审核 | 实物审核、行情评估、终审定价、生成代储订单 |
| 电子仓单管理 | 5种状态流转（待生效→普通存单→已质押→部分释放→已核销） |
| 保证金拨付 | 待拨付/已拨付状态管理、拨付确认、平安钱包支付 |
| 代储订单管理 | 已拨付订单管理、定向销售 |
| 销售订单管理 | 代储订单关联的销售订单查询（只读） |
| 出库指令管理 | 出库指令发布、指令状态管理 |
| 清结算中心 | 分账明细展示、详情查看 |
| 票据中心 | 发票列表展示、开具/下载（UI占位） |
| 提货凭证管理 | 提货码列表展示、状态查看 |

### 本期不包含（不做什么）

| 功能 | 说明 | 备注 |
|-----|------|------|
| 供应商侧申请入口 | 供应商通过醒市无仓提交申请，本期仅实现金融助手审核端 | 已在其他系统实现 |
| 仓储方WMS系统 | 复用现有仓储系统，本期仅做数据对接 | 外部系统 |
| 采购商下单流程 | 采购商通过市场助手下单，本期仅实现订单查看 | 已在其他系统实现 |
| 运营后台配置 | 费率配置、仓库配置等缺少运营后台 | 【待补充】 |
| 批量审批功能 | 已移除，改为单条审核 | V2.3明确移除 |
| 自动分账触发逻辑 | 仅展示分账结果，触发逻辑由清结算系统处理 | 外部系统 |
| 票据开具/下载功能 | 仅UI占位，需财务系统对接 | 【待补充】 |
| 提货码生成/核销 | 由销售订单支付和WMS系统自动处理 | 外部系统 |

## 关键决策记录

| 决策 | 原因 | 来源 |
|-----|------|------|
| 自动分账改为清结算中心 | 业务术语标准化，更符合清结算业务定位 | V2.3更新 |
| 审核页评估单价可编辑，货物总评估价只读 | 支持人工调整单品评估价，但总价必须自动计算保持一致性 | V2.3更新 |
| 去掉批量审批按钮 | 审核需要人工确认每笔货物，批量操作风险较高 | V2.3更新 |
| 仓储费按元/台/天配置 | 便于根据货物类型差异化定价 | V2.3更新 |
| 定向销售渠道默认仅市场助手可用 | 分销平台、购机通渠道暂未开放 | PRD初稿 |
| 出库指令由货款到账触发 | 系统实时监测支付状态，自动发布指令 | V2.3明确 |
| 销售订单页面只读 | 数据来源于市场助手系统，本期仅展示 | V2.3明确 |

## 待确认事项

| 事项 | 当前状态 | 建议确认方 |
|-----|---------|-----------|
| 运营后台配置功能 | PRD未提及，但费率、仓库等需要配置入口 | 产品+运营 |
| 行情系统对接方式 | PRD仅提及对接行情系统，具体接口未明确 | 技术+产品 |
| 平安钱包支付接口 | 拨付确认后具体调用方式未明确 | 技术+金融 |
| 销售订单数据来源 | 销售订单由市场助手产生，本期仅查询，实时性要求未明确 | 技术+产品 |
| 分账触发时机 | 清结算中心仅展示，触发逻辑由外部系统处理，但展示时效性未明确 | 产品+清结算 |
| 票据中心开具发票功能 | PRD仅列表展示，开具/下载功能未明确实现方式 | 产品+财务 |
| 页面加载/空状态UI | PRD未明确各页面的加载中、空数据状态展示 | UI设计 |
| 字段校验规则细节 | 如：保证金比例最高70%，但最低限制、步长未明确 | 产品+技术 |
| 出库指令触发方式 | 货款到账后自动触发还是需人工确认后发布 | 产品+技术 |
| 服务费率单位 | PRD写5‱（万分之5=0.05%），但示例计算评估总价 × 1%，存在矛盾 | 产品+财务 |

## 角色与权限

| 角色 | 权限范围 |
|-----|---------|
| 金融助手运营 | 代储代销审核、保证金拨付、代储订单管理、出库指令管理、清结算查看、票据中心查看、提货凭证查看 |
| 供应商 | 通过醒市无仓提交申请、查看拨付状态、查看结算明细（外部系统） |
| 采购商 | 通过市场助手下单、查看订单状态、使用提货码（外部系统） |
| 仓储方 | 通过WMS系统处理入库出库、核销提货码（外部系统） |

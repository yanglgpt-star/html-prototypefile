# Spec - 代储订单管理与定向销售

**对应需求点**：3.4 代储订单管理 + 3.4.4 定向销售弹窗  
**交付对象**：前端 + 服务端  
**依赖**：市场助手接口（创建销售订单）
**弹窗宽度**：900px

---

## Goal

金融助手运营能够管理已拨付的代储订单，并为代储订单创建定向销售订单。

**用户**：金融助手运营人员  
**操作**：查看代储订单列表 → 点击销售 → 填写销售信息 → 确认下单  
**达成**：创建销售订单，进入采购商下单流程

---

## Scope

### 包含
- 代储订单管理列表页（搜索、列表展示）
- 定向销售弹窗（销售渠道、提货方式、客户信息、商品信息）
- 销售订单创建提交

### 不包含
- 分销平台、购机通渠道（暂未开放）
- 采购商下单流程（市场助手系统）
- 档口自提的线下流程（仅记录提货方式）
- 添加地址功能的完整实现（仅UI占位）

---

## I/O

### 代储订单列表页接口

**搜索条件**
| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| orderNo | string | 否 | 订单编号 |
| supplierName | string | 否 | 供应商名称 |
| orderStatus | enum | 否 | 订单状态：PENDING_INBOUND/INBOUND/PAID/SELLING/COMPLETED |

**列表字段**
| 字段名 | 类型 | 说明 |
|-------|------|------|
| orderNo | string | 订单编号，如 DC20250909001 |
| supplierName | string | 供应商名称 |
| goodsInfo | string | 货物信息 |
| goodsQuantity | int | 货物数量 |
| actualInboundValue | decimal | 实际入库货值（元） |
| marginRatio | decimal | 保证金比例 |
| paymentAmount | decimal | 拨付金额（元） |
| paymentStatus | enum | 拨付状态 |
| inboundDate | date | 入库日期 |
| canSell | boolean | 是否显示"销售"按钮 |

### 定向销售弹窗

**获取代储订单信息（用于弹窗展示）**
| 字段名 | 类型 | 说明 |
|-------|------|------|
| orderNo | string | 代储订单编号 |
| supplierName | string | 供应商名称 |
| productName | string | 商品名称 |
| stockQuantity | int | 库存数量 |
| goodsList | array | 商品信息列表 |
| └─ category | string | 品类 |
| └─ brand | string | 品牌 |
| └─ productModel | string | 产品型号 |
| └─ spec | string | 规格 |
| └─ description | string | 产品说明 |
| └─ purchaser | string | 采购人 |

**客户列表（用于下拉选择）**
| 字段名 | 类型 | 说明 |
|-------|------|------|
| customerId | string | 客户ID |
| customerName | string | 客户名称 |

**地址列表（用于下拉选择）**
| 字段名 | 类型 | 说明 |
|-------|------|------|
| addressId | string | 地址ID |
| addressDetail | string | 地址详情 |

**业务归属人列表**
| 字段名 | 类型 | 说明 |
|-------|------|------|
| ownerId | string | 归属人ID |
| ownerName | string | 归属人姓名 |

**创建销售订单提交**
| 字段名 | 类型 | 必填 | 校验规则 |
|-------|------|------|---------|
| orderNo | string | 是 | 代储订单编号 |
| channel | enum | 是 | 销售渠道：MARKET（目前仅支持） |
| deliveryType | enum | 是 | 提货方式：EXPRESS/SELF_PICKUP |
| customerId | string | 是 | 下单客户ID |
| addressId | string | 是 | 收件地址ID（快递物流时必填） |
| businessOwnerId | string | 是 | 业务归属人ID |
| goodsList | array | 是 | 销售商品列表 |
| └─ skuId | string | 是 | SKU ID |
| └─ saleUnitPrice | decimal | 是 | 销售单价，必须 > 0 |
| └─ saleQuantity | int | 是 | 销售数量，必须 > 0 且 ≤ 库存 |

**异常情况**
- 销售单价 ≤ 0：提示"销售单价必须大于0"
- 销售数量 > 库存：提示"销售数量不能超过库存数量"
- 库存不足：提示"库存不足，当前库存：xx"

---

## Constraints

### 技术约束
- 销售渠道默认选中"市场助手"（蓝色高亮 #1890ff），分销平台和购机通选项置灰不可选（opacity: 0.5）
- 提货方式默认选中"快递物流"（蓝色高亮 #1890ff）
- 下单客户、收件地址、业务归属人均为下拉选择，必填标记红色星号
- 收件地址后显示"添加地址"按钮
- 商品信息表格字段：品类、品牌、产品型号、规格、产品说明、采购人、销售单价（输入框）、销售数量（输入框）
- 订单信息汇总使用蓝色信息卡（#e6f7ff）

### 业务约束
- 只有"已拨付"状态的代储订单才显示"销售"按钮
- 销售数量不能超过库存数量
- 销售单价由运营人员填写，无默认价格

## Acceptance Criteria

### 代储订单列表页
- ✅ 打开代储订单管理页面，展示代储订单列表
- ✅ 已拨付状态的订单显示"销售"按钮
- ✅ 其他状态（待入库、已入库、销售中、已完结）不显示"销售"按钮

### 定向销售弹窗
- ✅ 点击"销售"按钮，弹出定向销售弹窗
- ✅ 销售渠道区域：市场助手（默认选中，蓝色高亮 #1890ff）、分销平台（置灰，opacity: 0.5）、购机通（置灰，opacity: 0.5）
- ✅ 下单客户、收件地址、业务归属人字段标记红色星号（必填）
- ✅ 提货方式区域：快递物流（默认选中）、档口自提（可选）
- ✅ 下单客户下拉框展示可选客户列表（买家科技、华东采购、深圳数码）
- ✅ 收件地址下拉框展示可选地址列表，右侧显示"添加地址"按钮
- ✅ 点击"添加地址"按钮，提示"添加地址功能开发中"
- ✅ 业务归属人下拉框展示可选归属人列表
- ✅ 商品信息表格展示：品类、品牌、产品型号、规格、产品说明、采购人、销售单价（输入框）、销售数量（输入框）
- ✅ 订单信息汇总区域显示：代储订单编号、供应商、库存数量
- ✅ 点击"取消"，关闭弹窗，不创建销售订单
- ✅ 填写完整信息后点击"确认下单"，提示"销售订单创建成功"，关闭弹窗
- ✅ 销售数量输入 0，提示"销售数量必须大于0"
- ✅ 销售数量超过库存，提示"销售数量不能超过库存数量"
- ✅ 销售单价输入负数，提示"销售单价必须大于0"

# Spec - 代储代销审核

**对应需求点**：3.1 代储代销审核  
**交付对象**：前端 + 服务端  
**依赖**：行情系统接口（待确认）
**弹窗宽度**：1200px

---

## Goal

金融助手运营能够通过审核定价流程完成代储代销申请的终审，包括实物审核、行情评估、定价确认，最终生成代储订单。

**用户**：金融助手运营人员  
**操作**：查看申请列表 → 进入审核弹窗 → 查看货物明细 → 调整评估单价 → 填写定价参数 → 提交审核结果  
**达成**：生成代储订单编号，进入入库流程

---

## Scope

### 包含
- 代储代销审核列表页（搜索、列表展示）
- 审核定价弹窗（申请信息、货物明细、终审定价）
- 审核操作：审核通过、驳回修改、拒绝
- 详情查看弹窗（审核后的完整信息展示）

### 不包含
- 批量审批功能（已移除）
- 供应商申请提交入口（外部系统）
- 行情系统数据同步逻辑（假设行情数据已推送）
- 货物实物核验的线下流程（仅系统记录）

---

## I/O

### 列表页接口

**搜索条件**
| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| applyNo | string | 否 | 申请编号，支持模糊搜索 |
| supplierName | string | 否 | 供应商名称，支持模糊搜索 |
| auditStatus | enum | 否 | 审核状态：PENDING/PROCESSING/APPROVED/REJECTED/RETURNED |

**列表字段**
| 字段名 | 类型 | 说明 |
|-------|------|------|
| applyNo | string | 申请编号，如 SQ20250909001 |
| orderNo | string | 代储订单编码，审核通过后生成 |
| supplierName | string | 供应商名称 |
| goodsInfo | string | 货物简要信息 |
| applyQuantity | int | 申请数量 |
| applyAmount | decimal | 申请金额（元） |
| evalAmount | decimal | 核价金额（元） |
| auditStatus | enum | 审核状态标签 |
| applyTime | datetime | 申请时间 |
| canAudit | boolean | 是否显示"审核"按钮 |

### 审核弹窗接口

**获取申请详情**
| 字段名 | 类型 | 说明 |
|-------|------|------|
| applyNo | string | 申请编号 |
| orderNo | string | 代储订单编码 |
| supplierName | string | 供应商名称 |
| applyTime | datetime | 申请时间 |
| goodsList | array | 货物明细列表 |
| └─ brand | string | 品牌 |
| └─ productName | string | 商品名称 |
| └─ sku | string | SKU规格 |
| └─ skuCode | string | SKU编码 |
| └─ quantity | int | 数量 |
| └─ applyUnitPrice | decimal | 申请单价 |
| └─ applyTotalPrice | decimal | 申请总价 |
| └─ marketUnitPrice | decimal | 行情单价（来自行情系统） |
| └─ marketTotalPrice | decimal | 行情总价 |
| └─ evalUnitPrice | decimal | 评估单价（默认同行情，可编辑） |
| └─ evalTotalPrice | decimal | 评估总价（自动计算） |

**提交审核结果**
| 字段名 | 类型 | 必填 | 校验规则 |
|-------|------|------|---------|
| applyNo | string | 是 | - |
| evalUnitPrices | array | 是 | 每个货物的评估单价 |
| totalEvalAmount | decimal | 是 | 货物总评估价（只读，后端校验） |
| marginRatio | decimal | 是 | 保证金比例，范围 0-70，最多1位小数 |
| serviceFeeRate | decimal | 是 | 服务费率（‱），正整数 |
| validDate | date | 是 | 评估有效期，必须 ≥ 今天 |
| warehouseCode | string | 是 | 入库仓库编码 |
| storageFee | decimal | 是 | 仓储费（元/台/天），必须 > 0 |
| auditComment | string | 是 | 审核意见，最少5个字符，必填标记红色星号 |
| auditAction | enum | 是 | 操作类型：APPROVE/REJECT/RETURN |

**异常情况**
- 行情数据过期（>24小时）：提示"行情数据已过期，请重新获取"
- 评估单价超出范围（>行情单价 或 <成本价）：提示具体范围
- 重复提交：提示"该申请已处理，请勿重复提交"

### 详情弹窗接口

复用"获取申请详情"接口，增加审核后字段：
| 字段名 | 类型 | 说明 |
|-------|------|------|
| auditTime | datetime | 审核时间 |
| auditor | string | 审核人 |
| marginAmount | decimal | 保证金金额 |
| auditComment | string | 审核意见 |
| auditRecords | array | 审核记录列表 |

## Constraints

### 技术约束
- 使用现有金融助手前端框架（Vue3 + Ant Design Vue）
- 审核弹窗宽度 1200px，支持货物明细表格横向滚动
- 行情单价字段需高亮显示（蓝色 #1890ff）
- 必填字段标记红色星号（#ff4d4f）
- 货物明细表格共11列：品牌、商品、SKU、SKU编码、数量、申请单价、申请总价、行情单价、行情总价、评估单价、评估总价

### 业务约束
- 评估单价默认 = 行情单价，编辑范围：行情单价的 50%-100%
- 货物总评估价 = sum(评估单价 × 数量)，只读不可编辑，蓝色高亮 #1890ff
- 保证金金额 = 货物总评估价 × 保证金比例%，实时计算显示，绿色高亮 #52c41a
- 所有终审定价字段均为必填，标记红色星号
- 保证金比例范围 0-70%，最多1位小数
- 仓储费单位：元/台/天

### 状态流转
```
待审核 → 审核通过 → 生成代储订单
      → 驳回修改 → 回到供应商修改
      → 拒绝 → 流程结束
```

## Acceptance Criteria

### 列表页
- ✅ 打开代储代销审核页面，默认展示全部申请列表，按申请时间倒序排列
- ✅ 输入申请编号 SQ20250909001 搜索，列表只显示匹配结果
- ✅ 待审核状态行显示"审核"和"详情"两个操作按钮
- ✅ 已通过/已拒绝状态行只显示"详情"按钮，不显示"审核"按钮
- ✅ 列表分页：每页默认10条，支持切页

### 审核弹窗
- ✅ 点击"审核"按钮，弹出审核定价弹窗，宽度 1200px
- ✅ 申请信息区域显示：申请编号、代储订单编码、供应商、申请时间
- ✅ 货物明细表格显示 11 列：品牌、商品、SKU、SKU编码、数量、申请单价、申请总价、行情单价、行情总价、评估单价、评估总价
- ✅ 合计行显示：数量合计、申请总价合计、行情总价合计、评估总价合计，背景色 #fafafa，字体加粗
- ✅ 评估单价输入框默认显示行情单价，可编辑，修改后自动重新计算评估总价和货物总评估价
- ✅ 货物总评估价字段只读，显示为所有货物评估总价之和
- ✅ 保证金比例输入 70，保证金金额自动计算并显示为绿色（#52c41a）
- ✅ 保证金比例输入 80，提示错误"保证金比例最高不超过70%"
- ✅ 必填字段未填写时点击"审核通过"，提示"请填写完整信息"并定位到第一个未填字段
- ✅ 点击"审核通过"，确认弹窗后关闭审核弹窗，刷新列表，该申请状态变为"已通过"
- ✅ 点击"拒绝"，确认弹窗后关闭审核弹窗，刷新列表，该申请状态变为"已拒绝"
- ✅ 点击"驳回修改"，确认弹窗后关闭审核弹窗，刷新列表，该申请状态变为"待修改"

### 详情弹窗
- ✅ 点击"详情"按钮，弹出详情弹窗，展示完整的申请信息、货物明细、终审定价参数、审核记录
- ✅ 已通过的申请显示完整的审核时间、审核人、保证金金额等信息

---

## Non-Goals

- 不实现批量审批功能（已明确移除）
- 不实现供应商侧的修改申请功能（外部系统处理）
- 不实现行情系统的实时推送（假设数据已同步）
- 不实现线下实物审核的流程跟踪（仅记录结果）

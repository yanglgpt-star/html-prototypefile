# Spec - 销售订单管理

**对应需求点**：3.5 销售订单管理  
**交付对象**：前端 + 服务端  
**依赖**：市场助手系统（销售订单数据来源）

---

## Goal

金融助手运营能够查看代储订单关联的销售订单，跟踪订单的支付和发货状态。

**用户**：金融助手运营人员  
**操作**：查看销售订单列表 → 搜索/筛选  
**达成**：掌握代储货物的销售情况和订单履约状态

---

## Scope

### 包含
- 销售订单管理列表页（搜索、列表展示）
- 销售订单状态展示（支付状态、订单状态）

### 不包含
- 销售订单创建（由定向销售弹窗创建）
- 订单支付流程（市场助手系统）
- 订单发货流程（出库指令管理）
- 订单修改/取消操作

---

## I/O

### 列表页接口

**搜索条件**
| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| saleOrderNo | string | 否 | 销售订单编号 |
| buyerName | string | 否 | 采购商名称 |
| orderStatus | enum | 否 | 订单状态：PENDING_PAY/PAID/PENDING_SHIP/SHIPPED/COMPLETED |

**列表字段**
| 字段名 | 类型 | 说明 |
|-------|------|------|
| saleOrderNo | string | 销售订单编号，如 XS20250909001 |
| orderNo | string | 关联代储订单编号 |
| buyerName | string | 采购商名称 |
| supplierName | string | 供应商名称 |
| goodsInfo | string | 货物信息 |
| goodsQuantity | int | 货物数量 |
| saleUnitPrice | decimal | 销售单价（元） |
| actualPaidAmount | decimal | 实付金额（元），待支付显示"-" |
| paymentStatus | enum | 支付状态：PENDING/PAID |
| orderStatus | enum | 订单状态 |
| createTime | datetime | 下单时间 |
| payTime | datetime | 支付时间，待支付显示"-" |

### 数据说明

销售订单数据来自市场助手系统，本页面仅做查询展示，不维护订单状态流转。订单状态变化：
- 定向销售创建 → 待支付
- 采购商支付 → 已支付/待发货
- 出库指令发布 → 已发货
- 确认收货 → 已完成

---

## Constraints

### 技术约束
- 支付状态和订单状态使用标签展示，颜色区分
- 实付金额和支付时间，待支付状态显示"-"
- 列表共12列：销售订单编号、代储订单编号、采购商、供应商、货物信息、货物数量、销售单价、实付金额、支付状态、订单状态、下单时间、支付时间

### 业务约束
- 本页面为只读页面，不提供任何操作按钮
- 订单状态由市场助手和出库指令管理联动更新

---

## Acceptance Criteria

### 列表页
- ✅ 打开销售订单管理页面，展示销售订单列表
- ✅ 列表展示 12 列：销售订单编号、代储订单编号、采购商、供应商、货物信息、货物数量、销售单价、实付金额、支付状态、订单状态、下单时间、支付时间
- ✅ 待支付状态的订单，实付金额列显示"-"，支付时间列显示"-"
- ✅ 已支付状态的订单，实付金额显示具体金额，支付时间显示具体时间
- ✅ 支付状态标签：待支付（黄色 #fffbe6/#faad14）、已支付（绿色 #f6ffed/#52c41a）
- ✅ 订单状态标签：待支付（黄色）、待发货（橙色 #fff7e6/#d46b08）、已发货（蓝色 #e6f7ff/#1890ff）、已完成（绿色）
- ✅ 输入销售订单编号搜索，列表展示匹配结果
- ✅ 选择订单状态筛选，列表展示匹配结果
